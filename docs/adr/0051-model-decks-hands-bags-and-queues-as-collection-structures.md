# 0051 把牌库、手牌、袋与队列建模为集合结构

- 状态：已接受
- 日期：2026-07-20

## 决策

引入**集合结构（Collection Structure）**，用成员关系、重复性、顺序、访问方式、容量、所有权、可见性和允许变更描述对象怎样被组织。牌库、手牌、弃牌堆、抽取袋、栈、队列和背包等是这些维度的不同配置，而不是彼此独立的底层原语。

集合结构可以由加入、移除、移动、交换、洗混、排序、分组和抽样等规则改变。它不自动成为实体：具有独立规则身份、能够被单独追踪和改变的牌库可以是实体；“当前所有中毒敌人”之类的查询结果则可以是派生集合值。背包格等案例还可以同时组合规则空间及形状、容量或占据约束。

## 影响

分析集合相关机制时，应拆开记录组织维度和操作，不能用“有牌库／有背包”代替规则说明。这样能够比较卡牌、抽袋构筑、库存管理、单位编队和队列调度等跨媒介案例，同时保留它们在顺序、访问与信息结构上的关键差异。
