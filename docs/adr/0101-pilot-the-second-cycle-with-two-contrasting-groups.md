# 0101 用两个互补案例组试验第二轮方法

- 状态：已接受
- 日期：2026-07-22
- 来源决定：D2-1

## 决策

第二轮不立即铺开完整类型覆盖，先建立两个互补的**方法试验组**：

1. **连续行动与具身控制组**：检验动作／事件／过程、时间、空间、行动实现载体及输入反馈的边界；必做**重构测试**与**设计变体测试**，其他检验按主张触发。
2. **逻辑解谜与状态空间组**：检验局部一致、全局可扩展、不可逆动作、状态搜索与规则修改的边界；必做**重构测试**、**近邻辨别测试**与**独立标注测试**。

两个组都完成后进入[校准门 D](../../research/calibration-gates/gate-d-second-cycle-method-pilot.md)，先评估强检验方法本身是否可靠、可执行和具有区分力，再决定是否扩展到社会裁定、硬核模拟、持续世界等更昂贵的案例组。

本决定只冻结两个问题域及其主要检验职责，不冻结具体游戏、规则版本、组内角色、执行顺序或行为取证范围。具体案例必须另行决定，并在启动前完成事前检验命题和案例单位冻结。

## 依据

第二轮新增的重构、近邻、设计变体和独立标注方法本身还没有经过项目内校准。如果直接把它们施加到完整类型地图，字段负担、独立性污染或模糊通过标准可能在大量案例完成后才暴露。

两个先行问题域位于互补极端：连续行动具有实时、具身、连续状态和实现噪声；逻辑解谜通常具有离散、形式清楚、容易构造近邻与可达性反例的结构。若同一方法能在二者之间保持可重构性和区分力，才更有理由扩大应用范围。

这一选择同时回应项目的覆盖承诺：连续行动覆盖主流电子游戏中首轮明显缺失的平台、格斗、节奏等结构；逻辑解谜则继续深入拥有忠实玩家和独立设计传统的垂直类型。但两组只是方法试验，不代表已经完成这些类型的覆盖。

## 影响

- [第二轮校准协议](../../research/calibration-cycle-2-protocol.md)的状态改为“方法与先行组结构已接受，具体案例及顺序待定，尚未启动”。
- 第二轮首次校准门命名为**校准门 D**，在两组完成后集中决定方法修订、候选状态和后续扩展。
- 类型覆盖地图中的实时空间控制与逻辑解谜区域为候选来源，但原有案例名单不因本决定自动获选。
- 在案例冻结前不得启动完整取证或把候选游戏写成既定样本；已有移动试点材料可以作为选案证据，但不能代替第二轮的独立重构。
