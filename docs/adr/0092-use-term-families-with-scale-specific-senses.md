# 0092 用术语族管理定尺度义项

- 状态：已接受
- 日期：2026-07-21
- 来源决定：D-B5

## 决策

对“下注”“工人放置”等跨尺度行业名称采用“**术语族＋定尺度义项**”。同一表面名称可以索引动作、机制单元、复合机制、玩法模板候选和市场／社群类型标签，但每个义项都必须拥有明确尺度与稳定概念身份，并保存组成、编排或实例化关系。

本决定细化 [ADR 0055](0055-use-declared-scales-for-mechanic-boundaries.md) 与 [ADR 0057](0057-identify-gameplay-patterns-by-orchestration-signatures.md)：分析尺度继续决定对象边界，术语族只处理名称复用。

跨层复用的是名称，不是同一个设计对象在本体层级之间漂移。类型标签仍是历史、社群和市场索引，不能反过来决定形式模型中的层级。

## 影响

案例在首次出现实质歧义时声明当前义项；研究目录以后允许同名多尺度条目，但必须使用不同 ID。项目也不因一个著名名称主导整款游戏，就把局部机制自动等同玩法模板；当模板看似只含一个机制时，必须检查该机制内部是否已经提供持续活动、决策、反馈与目标循环。
