# 0063 以分级案例研究包组织正式案例

- 状态：已接受
- 日期：2026-07-20

## 决策

所有正式游戏案例使用统一的**案例研究包（Case Packet）**作为研究容器，而不在开放研究阶段直接写成书籍章节。完整结构包括案例身份卡、直观玩法说明、规则事实、机制拆解与比较、有类型编排图、玩家活动、玩法模板映射、实例化特征、边界与反例、设计变体以及证据账本。

案例包采用三级深度：快速案例只填写身份、核心拆解和一个有证据边界的结论；标准案例完成机制与玩法映射及主要来源；深度案例进一步完成冲突证据、反例、设计变体和生成实验。三级共享相同字段，使条目可升级、比较和跨章节复用。

## 影响

原理卷、实践卷与网页研究版都从案例包抽取所需材料，而不让早期目录决定研究顺序。典型主流案例可快速建立共同直觉，锚点／对照／纵深／边界案例再按研究问题提升深度；不要求为追求表面整齐而把所有游戏研究到同一程度。
