# 0059 为玩法模板记录证据状态

- 状态：已接受
- 日期：2026-07-20

## 决策

每个玩法模板记录**玩法模板证据状态（Pattern Evidence Status）**，区分：

1. **结构候选**：由规则、编排图或设计构想推导，尚无实际游玩证据。
2. **案例观察**：在具体游戏中有试玩观察、录像、策略资料、玩家记录或可靠研究支持相应玩家活动。
3. **跨案例支持**：多个不同游戏、媒介或设计传统显示相同编排不变量及可解释变体。
4. **生成性验证**：把模板用于机制替换、设计变体或原创原型后，预测的活动、决策与反馈实际出现。

这些是可并存、可修订的证据标签，而不是关于永恒真理、质量或流行度的认证。候选模板可以进入开放研究，但不得把规则允许的活动直接写成玩家已经采用的行为。

## 影响

玩法模板条目保存支持案例、玩家活动证据、反例或失效案例、适用边界和尚未验证的预测。经典案例用于建立共同直觉和解释力，跨案例与生成实验用于增强可复用性主张；发现玩家绕开预期循环时，应降低或改写相应主张，而不是只修改叙述以维护模板。
