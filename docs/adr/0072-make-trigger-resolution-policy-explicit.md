# 0072 显式记录触发结算策略

- 状态：已接受
- 日期：2026-07-21

## 决策

当多个规则实例可能同时或连续进入待结算状态时，机制语法使用**触发结算策略（Trigger Resolution Policy）**明确：在哪个状态快照检测触发；哪些规则实例组成候选集合；按同时、队列、栈、优先级、参与者选择或随机等何种方式排序；是否开放响应窗口；能否替换、阻止或取消；前序变化后是否重检；效果逐项还是整批提交；以及新触发、重复触发和无限连锁怎样终止。

触发结算策略与**效果时序**分开：前者决定多个待运行规则怎样获得执行顺序，后者决定已经结算出的状态更新怎样提交和被观察。不存在多项待结算规则的简单机制可以明确标记为不适用。

## 影响

“同时发生”“然后结算所有效果”等自然语言不再自动提供完整语义。死亡触发、反应链、连锁消除、实时争夺、速度排序和玩家选序等案例必须记录调度政策；文本顺序只有在规则明确赋予其优先级时才具有语义。
