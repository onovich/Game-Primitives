# 0084 在时间结构内记录调度语义

- 状态：已接受
- 日期：2026-07-21

## 决策

把**调度语义（Scheduling Semantics）**作为**时间结构**下的共享专业结构，记录驱动来源、更新频率或边界、阶段与优先级、同一时点的排序或并发、读取快照、暂停与积压，以及运行后重新检测的时点。

调度语义安排动作、事件与过程何时获得运行机会；**触发结算策略**处理多个已经满足触发的规则实例如何竞争与结算。它不增加“原语 → 机制 → 玩法模板 → 游戏”的公开层级，也暂不直接准入原语词汇。

## 影响

简单回合制案例可以由时间作用域隐含大部分调度信息；实时、自动、同时行动或并发系统必须按研究问题展开。模板不再只用“实时／回合制”代替时间模型，也不把程序源码的调用顺序未经解释便当作规则语义。
