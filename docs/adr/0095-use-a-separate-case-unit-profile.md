# 0095 使用独立案例单位剖面

- 状态：已接受
- 日期：2026-07-21
- 来源决定：D-C2

## 决策

案例研究使用独立的**案例单位剖面**，把研究涉及的下列对象绑定成可追溯身份图：

```text
游戏作品
  → 规则集版本
    → 游玩配置
      → 内容／材料实例
        → 游玩实例
          → 事件轨迹
```

这不是固定六级树。单位可以缺省、分叉或形成多对多关系；**内容／材料实例**、**游玩实例**和**事件轨迹**只在研究主张需要且证据可定位时建立。若某个题面、牌序、种子结果或初始布置只能在一次游玩开始时确定，可以把它记为游玩实例的初始状态，而不强行拆出独立单位。

每项证据主张都应标明实际适用的案例单位。规则来源可以支持规则集版本中的规范事实，却不能因此证明某个内容实例的唯一解、某次对局的行为、某场比赛的裁定过程或某条轨迹中的体验。

## 依据

数独规则家族与具体题面、《花火》的固定配置与具体牌序、《Factorio》的版本／地图／种子／运行、NetHack 的版本／配置／生成世界／存档，以及足球规则集与实际比赛都反复显示：作品标题和规则版本不是“一局游戏”的充分身份。

既有范围卡可以锁定版本，却没有统一表达这些对象之间的关系，也没有要求证据主张声明自己的真实主语。增加身份图能提高复现性和证据边界，而无需扩张公开理论主线。

## 影响

案例研究包模板升级为 v0.4，新增**案例单位剖面**，并把证据账本的“适用范围”改为“适用案例单位与范围”。已有十二案不要求机械回填；在补充具体题面、牌序、种子、存档、对局、比赛或轨迹证据时按需补建相应单位。

**案例单位剖面**属于研究身份与证据方法，不是**原语**、**机制**、**玩法模板**或**游戏**之间的新层级。
