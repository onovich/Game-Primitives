# 0057 以编排特征识别玩法模板

- 状态：已接受
- 日期：2026-07-20

## 决策

玩法模板的身份主要由**编排特征（Orchestration Signature）**决定，而不是由固定机制清单或类型标签决定。编排特征记录机制所承担的角色及其顺序、并行、循环、嵌套和触发关系，共享的状态、资源与信息，玩家反复进行的活动与关键决策，反馈和压力结构，局部与长期目标的连接，以及活动所在的时间尺度。

不同机制可以在保持这些关系时实现相似玩法模板，同一机制也可以在不同编排中服务不同玩法模板。玩法模板通常由多个机制共同形成，但如果单一机制自身已经稳定组织了反复的玩家活动、决策与反馈，也允许形成简单玩法模板，不为形式完整强制增加机制数量。

## 影响

玩法模板条目必须同时列出参与机制与编排关系，并说明哪些是身份不变量、哪些机制角色可以替换。设计分析不能以“机制 A + 机制 B”作为完整解释，也不能因两个游戏共享类型标签就推定它们共享同一玩法模板。
