# 0058 用有类型编排图表达机制到玩法模板

- 状态：已接受
- 日期：2026-07-20

## 决策

“原语——语法化——>机制”与“机制——编排——>玩法模板”采用两种相衔接但不同的形式工具。前者使用有类型规则槽位、条件、结算与状态转换，类似语法化的规则表达；后者使用**有类型编排图（Typed Orchestration Graph）**，描述多个已成立机制如何在运行中耦合并促成玩家活动。

编排图以机制、机制角色或玩家活动为节点，以时间、控制、状态、资源、信息、反馈、尺度与能动性关系为有类型边，并以行动、阶段、回合、关卡、整局或跨局等时间作用域限定关系。当前关系类型是开放、可修订的工作词汇，不声称已经完备。

## 影响

机制组合不能只写成加法清单，也不强行套用一条不断膨胀的规则句法。编排边仍须满足语义约束，例如资源存在接收方、信息确实进入相应观察视图、输出状态符合后续机制条件、触发者在对应时间作用域拥有权限。由此既保留形式严谨性，也承认玩法模板包含运行网络与玩家活动。
