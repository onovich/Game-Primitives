# 0055 以声明尺度划定机制边界

- 状态：已接受
- 日期：2026-07-20

## 决策

不追求一种跨所有研究问题绝对固定的“最小机制”。机制边界依赖明确声明的**分析尺度（Analysis Scale）**，但必须满足机制骨架并保留可追溯的拆解关系，不能任意把任何功能名称称为机制。

在选定尺度下，**机制单元（Mechanic Unit）**具有可评估的运行边界，能够说明触发或调用、条件与成本、动作或过程、结算以及可观察效果。多个机制单元稳定耦合、可在更高尺度作为整体调用、比较或修改时，称为**复合机制（Composite Mechanic）**；多个机制经由共享状态、资源、时间、信息和反馈形成多入口、循环或相互影响的持续网络时，称为**机制系统（Mechanic System）**。

## 影响

同一案例可以在教学概览中作为一个复合机制呈现，并在正式分析中放大为多个机制单元。目录条目必须说明当前尺度、边界依据、可继续拆分的部分、可替换或复用的子机制，以及原子提交和中断边界。机制单元、复合机制与机制系统仍属于公开主链中的“机制”，不增加新层级。
