# 0099 维护可逆的原语候选表

- 状态：已接受
- 日期：2026-07-22
- 来源决定：D-C6

## 决策

项目建立版本化的[原语候选表 v0.1](../../theory/PRIMITIVE-CANDIDATES.md)，统一登记候选的稳定编号、候选层级、声明尺度、语义职责、有类型接口、支持案例、反例、拆分／替换尝试、删除损失、设计变异预测、研究状态与复核条件。

候选表使用四种可逆状态：

- **暂准入**：跨异质案例具有稳定表达与设计收益，但仍可被反例撤销、拆分或合并；
- **保留候选**：出现重复压力，证据或边界尚不足；
- **降级**：改记为类型、参数、语义角色、复合结构、实现字段、比较剖面或来源术语；
- **拒绝**：当前原语主张已知会系统性混淆粒度、结构层或因果。

共享术语表与原语候选表分工维护：前者保证项目一致表达，后者承担准入审查。术语进入共享词汇不自动取得原语身份；“拒绝”也只拒绝当前粒度的原语主张，不禁止概念继续在适当层级使用。

第一轮仅在家族层暂准入若干基础候选。**暂准入**不表示已经证明绝对不可再分，也不允许把家族中列出的每个词自动扩写成获准原语。项目不在第一轮宣称候选表完整、最小、互斥或具有唯一分解。

## 依据

第一轮十二案反复需要实体与身份、状态事实、有类型关系、动作／事件／过程、规则适用与许可、结算与效果、时间、空间、集合、信息访问以及目标／评价／终止等基础表达家族。同时，校准已经证明“资源”“回合”“实时”“随机”“永久死亡”“工人放置”“身体”“承诺”和“例外”等熟悉词不能仅凭常见度取得同样身份。

若只维护共享词汇或名称列表，分析者可以通过不断增加词语使“任何游戏都可表达”，却无法证伪最小性、区分力与设计价值。候选状态、反例、拆分记录和复核条件使项目能够保留失败路径，并允许后续案例撤销当前判断。

## 影响

- `theory/PRIMITIVE-CANDIDATES.md` 成为候选状态的规范登记；校准报告中的候选表只作阶段摘要。
- 案例包可以提出候选与证据，但不能在单案内自行完成跨案例准入。
- `CONTEXT.md` 继续收录所有需要共享的术语，不再被误读为已经批准的原语表。
- “资源”继续按 [ADR 0089](0089-treat-resources-as-qualified-stackable-semantic-roles.md)作为可叠加语义角色；复合路径、剖面与实现字段保持非原语身份。
- 下一轮以重构、近邻辨别、设计变体、独立标注和反例测试复核暂准入家族；未通过者必须退回候选、降级或拒绝。
