# 0071 以五个层面表达动作结果

- 状态：已接受
- 日期：2026-07-20

## 决策

动作结果不以一个未注明含义的成功／失败布尔值表达，而以**动作结果（Action Outcome）**分别记录：输入是否被识别为规则动作；动作在当前状态是否合法；合法尝试的结算是成功、失败、部分成功还是未运行；实际提交了哪些效果或是否无状态变化；以及系统评价与玩家主观评价。

输入未识别、动作不合法、合法尝试失败、结算成功但效果为零、部分效果以及规则成功但策略不利是不同结果。规则成本可以在结算失败时仍然支付，玩家评价也不因系统标记奖励或惩罚而被规则保证。

## 影响

规则表达和案例分析必须注明“失败”发生在哪一层，以及被拒绝、重选、消耗成本、产生专门反馈、提交部分效果或保持无变化的具体后果。测试记录分别观察规则结果与玩家评价，不能用“成功”推断玩家获益或满意。
