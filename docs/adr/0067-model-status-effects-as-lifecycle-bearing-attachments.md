# 0067 把状态效果建模为具有生命周期的状态附着

- 状态：已接受
- 日期：2026-07-20

## 决策

以**状态附着（Status Attachment）**描述某项状态事实、关系或状态实例在一段时间内附着于目标，并通过规则修改条件、权限、求值、触发或效果。状态名称本身不是原语，其规则身份取决于实际携带的数据、修改的规则和生命周期。

状态附着可以只是一项简单状态事实，也可以具有来源、层数、强度、持续时间和独立身份，并包含施加条件、叠加／替换／刷新策略、生效规则、触发时点、衰减与到期、主动移除、来源归属和反馈，从而在较高尺度形成复合机制。

## 影响

中毒、流血、燃烧、眩晕、易伤、沉默和护盾等不逐项进入原语词汇。案例应展开其状态表示和规则生命周期：眩晕可能修改行动权限，燃烧可能触发周期效果，易伤可能修改伤害求值，护盾可能承担可消耗资源角色。相同名称在不同游戏中不得假定结构等价。
