# 0066 区分表达类型、内容标签与语义角色

- 状态：已接受
- 日期：2026-07-20

## 决策

正式区分三个维度：**表达类型（Expression Type）**规定某个值、实体、关系或表达式能够填入哪些规则槽位、被哪些操作符接受并产生什么输出；**内容标签（Content Tag）**是附着于内容类型、实体或状态、供规则查询的分类数据；**语义角色（Semantic Role）**说明某个对象或表达式在一次具体规则使用中承担 Actor、Target、Source、Destination、Cost、Reward、Observer 或 Owner 等职责。

实体类型属于表达类型的一种。同一实体可具有多个内容标签，也可在不同规则中承担不同语义角色。游戏自身使用“类型”一词时，项目不沿用其表面命名，而按实际功能判断它是表达类型、标签、枚举值还是普通状态。

## 影响

标签本身不产生行为，必须有规则谓词解释其效果；例如 Fire 或 Undead 只有进入克制、伤害或选择规则后才具有机制后果。机制语法用表达类型检查组合合法性，用语义角色说明当前职责，用内容标签承载可查询的领域分类，避免把三者压成一张含混“词性表”。
