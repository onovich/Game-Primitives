# 0097 记录行动实现载体、事件识别与裁定路径

- 状态：已接受
- 日期：2026-07-21
- 来源决定：D-C4

## 决策

机制分析按需记录**行动实现载体**与**事件识别与裁定路径**，以避免把所有游戏都压缩成“设备输入被程序立即写为状态变化”。前者是实现字段，后者是复合分析结构；两者都不录取为新**原语**，也不加入公开理论层级。

**行动实现载体**说明行动者的意图、选择或能力怎样通过身体、设备、口语、书写、物质操作、程序代理或混合媒介，在世界或规则执行系统中表现为可观察行为、材料变化、信号或输入。分析必须保留载体与行动者、受控实体、世界过程及规则识别的关系，不能只列控制器名称。

**事件识别与裁定路径**说明连续世界过程或离散发生怎样进入规则关注，成为候选事件，被观察或记录、分类、授权确认，写成正式规则事实，并触发效果、处罚、反馈或重启。阶段允许合并、缺省、分叉和复核；程序直接检测并提交的机制可以简写，不要求模拟人工裁判流程。

**动作生命周期**与**事件识别与裁定路径**可以共享里程碑，但观察方向不同：前者追踪行动从识别、提交到完成和退出，后者追踪已经发生的过程怎样取得正式规则身份。分析时通过里程碑 ID 互链，不将二者合并成万能流水线。

## 依据

FIDE 国际象棋已经表明，玩家身体、棋具、棋钟、对手和裁判共同实现规则；Game Boy《俄罗斯方块》则由程序解释输入、阻止非法候选并直接写回。足球进一步暴露数字输入模型的边界：运动员身体既实现行动，又是世界实体和规则作用对象；球与人的连续运动不会自动成为正式进球、越界或犯规，仍需观察、分类、裁定权与规则写回。

同样的压力预计会出现在体感、VR、节奏与格斗的身体输入、LARP、跑团口述、桌面材料操作、混合现实、传感器裁判和无障碍控制中。

## 影响

案例研究包模板 v0.4 在机制卡增加**行动实现载体**，并提供可选的**事件识别与裁定路径**表。简单软件机制可以显式简写；只有世界发生与正式规则事实分离时才展开完整路径。

项目不建立“身体原语”或“事件化层”。具身性、物质性与裁定不能被降为美术表现，但也不能仅凭媒介差异扩张公开本体。
