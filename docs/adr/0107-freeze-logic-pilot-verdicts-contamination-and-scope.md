# 0107 冻结逻辑解谜先行组的判定、污染与主张范围

- 状态：已接受
- 日期：2026-07-22
- 来源决定：D2-5c

## 决策

逻辑解谜先行组不使用加权总分或事后调整的及格线。每项测试先核对四个不可相互替代的硬条件，再给出一种**强检验结论**。术语字符串不同不自动构成失败；能否独立恢复结构区别才是本轮核心。

### 四项硬条件

| 编号 | 固定部分 | 隔离变化 | 必须恢复的区别 |
| --- | --- | --- | --- |
| `LC-01` | 4×4 值域、网格及行／列／区块约束 | 改变一项部分赋值 | 两个状态都满足已检查的**局部一致性**，但只有一个具有**全局可完成性** |
| `LC-02` | 位置、对象、推动许可与待判断动作 | 改变完成目标关系 | 同一推动在两案都满足**动作合法性**，但动作后只有一案仍能完成目标 |
| `LC-03` | 点阵、局部度数、无分支与无交叉条件 | 改变选择边的排列 | 两案都满足局部边条件，但只有一案形成单一连通环 |
| `LC-04` | 对象类型、规则识别方法与一次合法词块移动 | 改变词块位置 | 必须把普通对象状态变化与生效规则集合随之变化的**规则变换边界**分开 |

每项结果使用 `recovered`、`partially_recovered`、`missed`、`contradicted`、`not_evaluable` 或 `invalid`。`partially_recovered` 不计入硬条件通过；它保留为分析材料。

### 四种结论

| 机器值 | 共享表述 | 适用条件 |
| --- | --- | --- |
| `pass_in_scope` | **范围内通过** | 四项硬条件全部 `recovered`；两份有效独立标注在结构意义上一致；重构不依赖被隔离材料 |
| `model_failure` | **模型失败** | 程序与输入可解释，但至少一项硬条件被部分恢复、遗漏、合并或否定，或干净的独立标注在关键边界上不能复现 |
| `inconclusive` | **结论不定** | 输入歧义、任务说明缺口或污染使结果无法可靠归因给模型 |
| `run_invalid` | **轮次无效** | 冻结输入／承诺被覆盖、答案发生全局泄漏、承诺无法复算，或程序破坏到无法保留两份有效独立提交 |

**模型失败**只表示当前模型在该轮次、案例范围和硬条件下没有通过，不等于整套“原语 → 机制 → 玩法模板 → 游戏”假设被一次性推翻。精确术语不一致但结构区别一致，记录为词汇压力；只有区别本身无法稳定恢复才触发失败。

### 提交失效与轮次失效分开

单份提交若确认提前看见答案或另一份提交，标记为 `submission_invalid` 并永久保留。只要揭示尚未发生，且仍能重新取得协议要求数量的独立提交，轮次可以继续。只有答案向剩余测试者全局泄漏、冻结输入被修改、承诺不一致或无法补足有效提交时，才把整个轮次判为**轮次无效**。

## 污染记录

每份提交必须同时记录：

- 测试者类型、匿名标识、模型／人工身份和上下文范围；
- 既往项目暴露：`none`、`limited`、`substantial`、`unknown`；
- 原型识别：`none`、`suspected`、`identified`，以及识别出的家族；
- 外部检索、所用来源及范围状态：`not_applicable`、`within_scope`、`outside_scope`、`unknown`；
- 包外文件访问及路径；
- 他人提交暴露与答案暴露：`none`、`suspected`、`confirmed`；
- 任何无法归入上述字段的说明。

既往熟悉或认出原型只把证据标为 `downgraded`，不自动删除结果；确认接触答案或他人提交使对应提交失效；仅有怀疑而无法查明时，结论原则上降为**结论不定**。所有不利污染均保留，不能通过换测试者后删除旧记录来美化结果。

## 行为主张边界

本轮 `behavior_scope` 固定为 `structural_only`。测试只判断规则、状态、作用域、许可、可满足性／可达性与规则集合变化，不主张真实玩家通常怎样推理、是否会预见死路、哪种解法最佳、哪个案例更难或产生何种体验。

若提交或报告出现玩家策略、难度、平衡或体验陈述：

1. 将其标记为范围外主张，不计入硬条件；
2. 能撤回则撤回；仍需研究则移交[校准证据积压](../../research/calibration-evidence-backlog.md)；
3. 若整体结论必须依赖该主张，则本轮不能判为**范围内通过**。

因此逻辑组启动条件中的行为取证边界已经冻结为“不在本轮提出行为主张”，而不是伪造一套玩家观察材料。

## 制品约束

[逻辑先行组测试工作区](../../research/calibration-tests/logic-pilot/README.md)增加四类 Schema：

- 输入包 Schema 只约束隔离信封、随机顺序和载荷模式标识，不把某种尚未验证的通用游戏结构偷偷写进格式；
- 提交 Schema 保存回答、测试者身份范围和完整污染记录；
- 真值 Schema 保存揭示后的内部映射、四项硬条件与预期判断；
- 报告 Schema 保存逐项结果、污染影响、承诺验证和四类总体结论。

具体网格、图、状态或规则词块载荷仍须在制作 `logic-001` 输入时分别定义小型 payload Schema；这属于测试制品制作，不新增理论层级。
