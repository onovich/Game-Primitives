# 0060 以多对多实例化特征连接玩法模板与游戏

- 状态：已接受
- 日期：2026-07-20

## 决策

把“玩法模板——实例化——>游戏”理解为具体化过程，并以**实例化特征（Instantiation Profile）**记录角色绑定、数值与概率参数化、卡牌／单位／关卡／事件等内容化、多个模板的组合、输入与反馈的交互映射、软件／实体材料／裁判或协商等媒介实现，以及人数、模式、局长、平台和社会关系等情境限定。

玩法模板与游戏是多对多关系：一个模板可以产生许多具体游戏，一个游戏通常也同时实例化多个主次不同的模板。实例化不是简单换皮，也不是把一张模板配方一对一套入作品。

另以**游玩实例（Play Instance）**指特定参与者在具体规则版本与情境下实际发生的一次对局、会话或持续游玩过程。主链中的“游戏”指可供游玩的具体设计系统或作品版本，不指一次实际游玩。

## 影响

游戏案例应说明抽象角色如何落到具体内容、数值、界面、媒介和情境，并列出共同存在的玩法模板。玩家活动证据应附着于游玩实例、观察记录或可核对材料，不能把设计系统允许的行为直接当作已经发生的事实。
