# 0068 把规则修改器建模为有类型高阶结构

- 状态：已接受
- 日期：2026-07-20

## 决策

以**规则修改器（Rule Modifier）**描述对既有规则、槽位或有类型表达进行增加、替换、缩放、限制、插入、禁用或覆盖的高阶规则结构。每个修改器必须声明来源、目标、具体操作、参与者／实体／空间／时间作用域、激活条件、生命周期、与其他修改的优先级和叠加策略。

这是语义模型，不要求实现真的动态改写程序规则；通过读取状态和计算派生结果实现，只要观察语义一致，也可表达为同一修改。无类型的“随机、隐藏、自动、持续、双倍”等不作为宽泛修饰词原语：它们分别落到结算、信息访问、能动性或触发、时间过程以及特定数值表达等明确位置。

## 影响

装备、状态、科技、角色能力和临时事件对规则的影响可以用统一结构比较。案例必须说明修改了哪条规则或哪个槽位，以及冲突、叠加与撤销方式；不能用 `Random + Attack`、`Hidden + Card` 等关键词拼接代替机制表达。
