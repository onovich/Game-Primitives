# 0061 把游戏案例结论绑定到正确研究单位

- 状态：已接受
- 日期：2026-07-20

## 决策

正式游戏案例区分四个研究单位：**游戏作品（Game Work）**是跨版本、由产品身份、设计传统或玩家文化识别的作品或规则传统；**规则集版本（Ruleset Version）**是在明确版次、补丁、日期或规范来源下适用的规则、参数与内容；**游玩配置（Play Configuration）**进一步限定模式、人数、难度、扩展、地图、牌池、赛事格式、家规或服务器设置；**游玩实例（Play Instance）**是在该版本与配置下实际发生的一次对局或会话。

每项结论必须绑定到能够支持它的单位。规范规则主张引用规则集版本，模式主张引用游玩配置，玩家实际行为引用游玩实例或聚合行为证据，跨版本身份与历史主张才上升到游戏作品层面。

## 影响

电子游戏案例记录平台、版本或补丁及资料日期；桌游记录版次、扩展和规则文档；传统游戏记录采用的规则机构或规范；家规与社群变体记录规则地位。不能用一个宽泛作品名掩盖版本差异，也不能从某次对局直接推断整个作品的普遍玩法。
