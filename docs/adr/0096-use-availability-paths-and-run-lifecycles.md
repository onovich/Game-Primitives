# 0096 使用可用化路径与运行生命周期

- 状态：已接受
- 日期：2026-07-21
- 来源决定：D-C3

## 决策

项目接纳**可用化路径**与**运行生命周期**为可选复合分析结构。两者都由既有状态、关系、机制、时间、许可、裁定和效果组合而成，不录取为新**原语**，也不加入公开理论层级。

**可用化路径**记录一个构件、能力或行动资格从取得、解锁或生成，到能够被访问、部署、供给、激活并实际生效所经过的状态转换与门控。它不是固定线性流程；案例可以跳步、分叉、循环、回退或失效。只有取得与生效确实分离，且中间状态对研究问题有解释价值时才展开。

**运行生命周期**围绕明确声明的游玩或运行单位，记录开始、持续、暂停、恢复、候选终止、拦截例外、授权裁定、锁定、结果披露与跨实例后效。若一手、一轮、一局、一次运行、一场比赛、会话或战役相互嵌套，分别声明对象，不能用一个“结束”包办所有尺度。

## 依据

《Dominion》中新取得的牌通常还要经过弃牌、洗牌、抽取与阶段资格才可打出；《Factorio》中的建筑物品还需放置、连接、供能、供料并保有输出容量才产生预期效果。《农场主》的家庭成员资格与《星际争霸》的单位生产也支持“拥有／解锁不等于当前可用”。

国际象棋、德州扑克、NetHack 与足球则显示：终止可能有不同对象、候选事件、拦截规则、裁定者、锁定点、恢复资格、披露和跨实例后效。把“将死”“死亡”“弃牌”“终场”写成同一瞬时终止动作会丢失关键设计差异。

## 影响

案例研究包模板 v0.4 在**目标、终止与评价**下增加可选**运行生命周期**，在机制编排下增加可选**可用化路径**。模板允许省略不适用阶段，并禁止为了填满路径而虚构规则门控。

已有案例不机械回填；补充具体对局、运行、比赛或设计变体时，只在相关中间状态会改变合法性、可行行动、反馈或证据结论的情况下展开。
