# 0102 先执行逻辑解谜组，再执行连续行动组

- 状态：已接受
- 日期：2026-07-22
- 来源决定：D2-2

## 决策

第二轮两个方法试验组按以下顺序执行：

```text
逻辑解谜与状态空间组
    ↓ 组间方法复核
连续行动与具身控制组
    ↓
校准门 D
```

先用形式边界较清楚的逻辑解谜材料检查**重构测试**的输入隔离、**近邻辨别测试**的判别命题和**独立标注测试**的分歧记录。逻辑组完成必做测试后进行一次组间方法复核；该复核只修补执行含混、材料泄漏和不必要负担，不提前代替校准门 D 作理论结论。

连续行动组随后使用复核后的方法，检验它在连续时间、具身输入、实现噪声和反馈回路中的表现。两组都完成后才召开校准门 D，避免只凭形式清晰的案例判断方法有效。

本决定不冻结任何具体游戏、谜题、版本或内容实例。第二轮仍未启动；逻辑组必须先完成核心检验命题、案例角色和材料隔离方案的决定。

## 依据

独立重构和标注方法若在规则边界清晰的离散案例中仍然产生大量不可解释分歧，说明问题更可能来自术语、模板或测试说明；此时直接进入连续行动，只会把方法缺陷与实时实现复杂性混在一起。

相反，只在逻辑解谜中通过也不足以证明方法适用于电子游戏动作系统。第二步保留连续行动组，正是为了让经过初步校准的方法面对不同的时间、空间、输入和实现条件。

## 影响

- [第二轮校准协议](../../research/calibration-cycle-2-protocol.md)和[校准门 D](../../research/calibration-gates/gate-d-second-cycle-method-pilot.md)记录固定执行顺序。
- 下一项决策转为冻结逻辑解谜组的核心检验命题与案例角色。
- 连续行动组可以保留候选池和准备性问题，但在组间方法复核前不启动完整取证与强检验执行。
- 组间复核的任何改动必须留下版本记录；校准门 D 仍负责比较两组后作正式方法决定。
