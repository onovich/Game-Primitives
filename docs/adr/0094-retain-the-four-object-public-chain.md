# 0094 保留四对象公开主线

- 状态：已接受
- 日期：2026-07-21
- 来源决定：D-C1

## 决策

第一轮十二案校准后，项目保留下面这条面向读者的公开主线：

```text
原语 --语法--> 机制 --编排--> 玩法模板 --实例化--> 游戏
```

**原语**、**机制**、**玩法模板**与**游戏**是四类主要设计对象；**语法**、**编排**与**实例化**是对象之间的组合关系，不作为三个额外层级。主线是一把可双向使用的设计缩放尺，而不是游戏必须遵循的创作顺序或生产流程。

版本、配置、内容实例、游玩实例、事件轨迹、证据状态、动作生命周期、调度、测量、实现载体和社会情境可以作为研究身份、内部结构或压力测试维度，但不能仅因分析有用就进入公开主线。未来若要增加公开对象或层级，必须展示当前四对象与三关系无法消除的跨案例失真，而不是只证明新概念值得记录。

## 依据

国际象棋、Game Boy《俄罗斯方块》、无限注德州扑克、《农场主》、数独、《花火》《Dominion》《Factorio》《星际争霸》《外交》、NetHack 与足球没有要求增加公开层级。案例暴露的重复问题都可以先在规则内部结构、编排关系、实例化信息或研究证据中准确表达。

这不证明主线已经普适、完备或不可修订。决定只表示：在第一轮声明范围内，增加层级带来的复杂度没有得到相应解释收益。

## 影响

`theory/FOUNDATIONS.md` 不再把**语法**与四类设计对象并列为“五个层级”，并明确补入**编排**与**实例化**的关系身份。共享词汇、案例模板和后续书稿必须区分：公开对象、组合关系、对象内部结构与研究元数据。
