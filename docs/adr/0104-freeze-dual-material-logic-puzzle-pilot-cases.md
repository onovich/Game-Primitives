# 0104 冻结逻辑解谜先行组的四个锚点与受控夹具

- 状态：已接受
- 日期：2026-07-22
- 来源决定：D2-4

## 决策

**逻辑解谜与状态空间组**采用“**锚点案例＋受控测试夹具**”双材料设计。每个案例角色同时保留：

1. 一份有明确来源、作品身份与内容定位的经典锚点，用于检验模型能否重构真实作品；
2. 一份由项目制作、去除作品名与表现身份的最小夹具，用于隔离变量、盲化重构与独立标注。

四种角色冻结如下：

| 案例角色 | 锚点内容实例 | 受控测试夹具基线 |
| --- | --- | --- |
| **静态约束锚点** | Nikoli 英文数独页的例题、途中图与答案图 | `FX-L01 v0.1`：4×4 局部一致但可扩展／不可扩展状态对 |
| **不可逆状态空间对照** | 《仓库番》权利方规则页的 5×5 示意盘面 | `FX-L02 v0.1`：同一合法推动通向完成／不可恢复状态对 |
| **局部—全局结构纵深** | Nikoli 英文 Slitherlink 页的例题、途中图与答案图 | `FX-L03 v0.1`：局部条件相同的单环／双环边集对 |
| **规则变换边界** | Hempuli 2017 NGJ 作者参赛包 upload `455759` 中的 `1level`，以同包 `syntax.lua` 为执行基准 | `FX-L04 v0.1`：词块移动前后规则集合改变的匿名状态对 |

[一手来源选案笔记](../../research/sources/second-cycle-logic-pilot-case-selection.md)中的稳定链接、文件长度、SHA-256、配置、观察范围、污染和开放缺口构成本决定的来源基准。官方图片、程序包、源码和商业关卡文件不在仓库重新分发；网页与公开测试包通过权利方链接和哈希复核锚点，只直接发布项目夹具。

本决定冻结 `FX-L01`–`FX-L04` 的职责和最小内容，不声称机器可读文件已经存在。夹具文件、统一格式、版本哈希、可见／隔离材料、通过标准和污染登记仍须在测试启动前完成。

## 依据

只使用经典作品会让熟悉案例的重构者凭游戏常识补全缺失信息；只使用项目自制夹具，又可能使模型只对过度干净的人造状态成立。双材料把“面对真实作品的完整性”与“隔离变量的检验力”分开，同时保留两者之间的对照。

一手来源核查还排除了两个看似方便但不够稳定的方案：

- 不用持续更新的商业版《Baba Is You》关卡，改用作者发布、内容和执行实现均可校验的 NGJ17 原始参赛包；
- 不把非权利方网站转录的 `Original & Extra #1` 当作《仓库番》一手关卡，改用权利方规则页示意盘面承担锚点。

这些选择提高的是版本、来源和盲测边界的可靠性，不证明四个作品已经覆盖全部逻辑解谜传统。
