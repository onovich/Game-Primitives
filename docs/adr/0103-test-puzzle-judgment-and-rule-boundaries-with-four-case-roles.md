# 0103 用四种案例角色检验解谜判定与规则边界

- 状态：已接受
- 日期：2026-07-22
- 来源决定：D2-3

## 决策

**逻辑解谜与状态空间组**冻结以下可反驳命题：

> 单一的“合法”或“约束”谓词不足以无损重构逻辑解谜；结构描述至少必须使分析者能够区分**动作合法性**、**局部一致性**与**全局可完成性**，并能另外表达判定规则本身成为动作对象的**规则变换边界**。

前三项不是难度等级，也不是同一判定的三种说法：

- **动作合法性**询问某一动作此刻是否被规则允许；
- **局部一致性**询问当前状态在声明的局部检查范围内是否尚无冲突；
- **全局可完成性**询问当前状态是否仍存在满足全部完成条件的扩展或合法路径。

**规则变换边界**不是“第四种合法性”。它承担边界压力：动作效果可能改变后续用于判定条件、许可与效果的**规则**或**规则修改器**。本决定要求模型保留这种差异，但不把四项分别录取为**原语**，也不预先决定静态可满足性与动态可达性是否应该共用一个底层候选。

逻辑组据此冻结四种案例角色：

| 案例角色 | 检验职责 |
| --- | --- |
| **静态约束锚点** | 检验部分状态没有当前冲突时，是否仍需另外表达完整扩展的存在性 |
| **不可逆状态空间对照** | 检验一个当前合法动作能否把系统推进到无法完成或无法恢复的状态 |
| **局部—全局结构纵深** | 检验局部条件均成立时，是否仍需独立的全局拓扑或整体结构条件 |
| **规则变换边界** | 检验玩家操作改变生效中的规则表达时，普通状态更新是否足以重构后续判定 |

这些是结构职责，不是已经选定的游戏名单。数独、推箱子、数回与《Baba Is You》目前只是最直观的候选示例；D2-4 仍须分别冻结作品、规则版本、配置、内容实例、观察范围和来源基准。

## 可反驳条件

当前命题在以下任一情况下必须拆分、缩小或撤回：

1. 单一有类型表达在四种角色中仍能通过独立重构与近邻辨别，而不依赖游戏常识补全；
2. 两个独立标注过程无法稳定识别任何对应差异，且分歧不能归因于说明或材料污染；
3. 某种案例角色没有隔离预定变量，实际只重复另一角色；
4. 所谓差异只影响求解策略、难度或体验，却没有规则结构差异且缺少行为证据。

## 依据

第一轮数独案例已经表明“没有当前冲突”不自动推出“仍存在完整解”，由此形成原语候选 `GP-C005`。但单一案例不能判断这究竟是独立原语、一般可满足性／可达性判断、语法作用域，还是描述模板遗漏。

四种角色把这一失败压力逐步移位：从静态赋值到不可逆转换，从局部条件到全局结构，再到规则本身可被改变。若同一组共享区别能够跨越这些角色并通过强检验，候选才获得更强支持；若区别随案例类型消失，就应缩小或拆分，而不是继续增加抽象词。
