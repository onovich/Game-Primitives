# 0078 把体验主张写成情境化假设

- 状态：已接受
- 日期：2026-07-21

## 决策

“有趣、紧张、爽快、沉浸、挫败、掌控感”等不作为机制属性或公开主链的新层级，而写成**体验假设（Experience Hypothesis）**：关于某种机制、玩法模板及其实例化方式会在指定玩家群体与游玩情境中促成何种主观体验的可检验预测。

完整假设注明目标玩家或差异、规则与编排原因、实例化条件、预期玩家活动、预期体验、行为指标、自我报告或访谈方式及可能出现的相反体验。设计意图、体验预测、实际行为、玩家自我报告和研究者解释分别保存，不能互相替代。

## 影响

案例和练习不得直接声称永久死亡必然紧张、随机必然刺激、隐藏信息必然促成推理或反馈必然带来掌控感。体验假设附着于机制、编排、实例化和游玩实例，通过行为与体验证据修订；相反体验和玩家差异是模型边界，不作为应被删去的噪声。
