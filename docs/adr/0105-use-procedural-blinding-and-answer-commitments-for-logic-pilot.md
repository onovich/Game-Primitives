# 0105 逻辑解谜先行组采用程序性盲化与答案承诺

- 状态：已接受
- 日期：2026-07-22
- 来源决定：D2-5a

## 决策

**逻辑解谜与状态空间组**采用可审计的**程序性盲化**。它不把公开仓库伪装成安全沙箱，而是用角色分离、最小可见材料、不可覆盖的**首次提交**、事后揭示和污染声明降低答案泄漏。

先行组区分三种分析角色，另设一个不参与当前判断的操作角色，避免把两个相反方向的工作混为一谈，也避免让答案无人保管：

| 角色 | 允许看见 | 必须隔离 |
| --- | --- | --- |
| **来源编码者** | 冻结的一手来源、版本与哈希、案例范围、测试本地允许术语表、输出格式 | 既有案例分析与项目结论、D2-3 的预期判别、夹具映射、答案包和下游提交 |
| **重构者** | 已冻结的结构描述、声明范围、必要类型说明、测试本地允许术语表和任务格式 | 作品身份、原规则与官方图、来源编码过程、既有项目分析、答案包、夹具和其他提交 |
| **夹具标注者** | 匿名且乱序的规则、状态和任务输入，测试本地允许术语表与输出格式 | 作品身份、案例角色、`FX-L01`–`FX-L04` 映射、预期标签与理由、锚点材料、答案包和其他提交 |
| 测试保管者 | 答案包、随机映射、`secret_nonce`、承诺与提交校验值 | 不得参加本轮来源编码、重构或夹具标注；承诺建立后不得覆盖答案包 |

第一种工作称为**来源编码**：它把“来源事实”转写为“结构描述”。只有第二种工作才是**重构测试**：它从结构描述反向恢复关键规则。来源编码者不得同时充当同一材料的盲化重构者。

公开仓库中的既有文档无法从技术上对测试者隐藏。正式测试只向测试者指定隔离输入包，并要求不读取包外项目文件；测试者在提交后声明此前是否接触过相关文档、是否识别出原型家族、是否使用外部资料。已知暴露保留为结果字段，不得静默删除不利提交。

## 答案承诺与揭示顺序

每个待揭示答案包在测试前建立**答案承诺**：

```text
commitment = SHA-256(secret_nonce_bytes || exact_truth_bundle_bytes)
```

其中 `secret_nonce` 是测试前生成、揭示前不公开的 32 字节随机数；散列对象是后续实际公开的精确答案包字节。公开承诺记录至少包含算法、协议／模式版本、摘要和建立时间，但不包含随机数或答案。答案包与随机数由测试保管者保存在仓库外；揭示前均不得进入公开 Git 历史或测试者可见目录。

本轮需要独立判断的标注任务，在揭示前至少取得两份彼此不可见的**首次提交**。来源编码输出必须先于重构输入冻结；重构者与夹具标注者在各自首次提交前不得看见其他提交。首次提交形成时间与 SHA-256 后不可覆盖，格式勘误或后续共识只能追加。

只有预先要求的**首次提交**全部冻结后，项目才依次公开：

1. 精确答案包和 `secret_nonce`；
2. 承诺复算结果；
3. 逐项比较与分歧记录；
4. 熟悉度、原型识别、额外检索和意外暴露等污染声明。

承诺复算成功只能证明答案包没有按提交结果事后改写，不能证明答案正确、任务设计无偏或理论成立。

## 依据

双材料设计同时面临两种相反风险：**锚点案例**容易被游戏常识补全，**受控测试夹具**又容易让研究者按预期标签制作和解释。角色分离让来源事实、结构描述、反向重构和匿名标注留下不同制品；**答案承诺**则防止看到结果后移动判定基准。

真正的安全隔离需要独立账户、文件权限或外部参与者，本项目当前没有该条件。把限制准确称为**程序性盲化**，比声称“绝对盲测”更符合公开研究仓库的实际边界。
