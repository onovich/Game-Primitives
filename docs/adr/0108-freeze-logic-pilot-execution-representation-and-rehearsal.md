# 0108 冻结逻辑先行组的执行、表示与流程彩排

- 状态：已接受
- 日期：2026-07-23
- 来源决定：D2-6a–q

## 决策

`logic-001` 在正式收集前先冻结执行角色、独立性边界、受测表示、输入规模、提交格式、流程彩排和人工放行门。该决定完成测试执行方法，不表示正式测试已经开始。

## 角色与最低配置

主编线程只担任**测试保管者**，持有真值、随机映射、`secret_nonce`、公开承诺和提交校验值，不参加本轮来源编码、重构或夹具标注。

最低执行配置固定为 `1＋1＋2＋2`：

| 工作 | 数量 | 模型与参数 | 职责 |
| --- | ---: | --- | --- |
| **来源编码者** | 1 | `gpt-5.6-sol`，`high` | 把冻结的一手来源转写为 GP-SR 0.1 结构描述 |
| 来源忠实度审核者 | 1 | `gpt-5.6-terra`，`high` | 只核对来源事实、遗漏、歧义、身份泄漏与越界 |
| **重构者** | 2 | `gpt-5.6-sol`，`high` | 从相同冻结描述独立恢复关键规则 |
| **夹具标注者** | 2 | `gpt-5.6-sol`，`high` | 对相同匿名夹具独立作结构判断 |

来源编码与审核故意使用不同模型，减少共同遗漏；两份重构和两份标注分别使用同一模型、同一参数和不同空白会话，控制执行者能力变量。模型身份仍进入每份提交，不能把同模型的独立运行外推为跨模型或人类一致性。

## 独立性的边界

本轮使用**独立执行**，不是安全沙箱或统计意义上的独立人群样本。每名盲测执行者从不继承项目历史的空白会话启动，只收到同一冻结输入，在首次提交前看不到其他提交。

2026-07-23 的金丝雀测试得到：

- 一个空白会话不知道只在另一会话提示中出现的 `CHAT_CANARY`；
- 同一空白会话能够读取另一会话写入共享工作区的 `FILE_CANARY`。

因此会话上下文可以程序性隔离，而协作代理的文件系统完全共享。另建普通 workspace 或 Git worktree 只能降低误读概率，不能形成权限边界。本轮采取以下补偿：

1. 重构与标注输入直接随任务派发，不要求读取仓库；
2. 明确禁止盲测执行者调用工具或读取包外材料；
3. 真值、密钥和其他提交在预定提交收齐前不以明文进入共享工作区；
4. 包外访问、既往熟悉与原型识别继续进入污染声明；
5. 报告明确承认文件读取违规目前不能被技术性排除或完整审计。

该方案只支持**程序性盲化**。若以后需要安全级隔离，必须另用受限 API、容器、独立系统账户或外部人工执行。

## 来源忠实度审核

来源编码者与审核者不以多数票、当前理论偏好或“哪份更容易通过测试”裁决分歧。审核异议必须绑定冻结来源定位，并归入事实错误、必要规则遗漏、歧义、身份泄漏或范围越界。编码者修订后由审核者复核；无法由来源裁定的实质分歧会阻止对应案例冻结。

两者均不得看见 `LC-01`–`LC-04` 的预期判断、匿名夹具映射、真值或下游提交。来源编码结果使用四种状态：

| 机器值 | 含义 | 结果作用 |
| --- | --- | --- |
| `encoded` | 来源事实已经忠实进入受测表示 | 可进入下游测试 |
| `source_underdetermined` | 来源本身不足以确定该事实 | 对应项目不可评价；必要覆盖缺失时为**结论不定** |
| `representation_gap` | 来源事实明确，但 GP-SR 0.1 无法承载 | 直接形成**模型失败**证据，其余项目继续用于定位 |
| `scope_excluded` | 事前明确排除在本轮范围外 | 仅在不移除硬条件时可继续 |

不得为消除 `representation_gap` 在运行中新增槽位或改写表示。

## 受测表示与术语边界

本轮只测试[游戏原语结构表示 v0.1](../../theory/STRUCTURAL-REPRESENTATION-0.1.md)，不测试尚不存在的完整 DSL，也不把整个项目理论作为一个不可分割对象接受或否定。

测试本地术语表允许**实体、状态、关系、动作、前置条件、效果、目标、作用域、时间、信息、规则生效**等基础结构词；在盲测输入与说明中禁止**局部一致性、全局可完成性、规则变换边界**、案例角色和 `LC-01`–`LC-04`。来源专名与作品术语先映射为中性名称或匿名 ID；原始外来术语只留在来源编码与审核材料。

测试者不因没有复述项目术语而失败。结构区别一致、措辞不同只形成词汇压力。

## 输入与提交

正式材料固定为：

- 4 个重构项目：四个经典锚点的冻结结构描述；
- 8 个标注项目：`FX-L01`–`FX-L04` 各自的 A／B 状态；
- 首轮不增加诱饵题或重复题。

重构包和标注包各随机排列一次，然后分别冻结；同类两名执行者看见完全相同的内容与顺序。项目使用无语义 ID，A／B 不显示配对关系且不相邻；随机种子、真实映射和配对关系只进入密封真值。

提交采用结构化 JSON 与自然语言说明并存的混合格式。每项必须记录结构判断、依据、假设和歧义，并允许 `indeterminate`；置信度只用于诊断，不进入通过判定。第一份 JSON 原样冻结，后续只能追加勘误。

## 冻结纪律

`logic-001` 只有在以下测试者可见制品全部生成、校验、提交并推送后，才从 `preparing` 进入 `frozen`：

- GP-SR 0.1 规范；
- 任务说明与测试本地术语表；
- 经审核的来源编码；
- payload Schema；
- 随机化输入及全部 SHA-256；
- 密封真值对应的公开**答案承诺**。

首个 `frozen` Git 提交以后，任何测试者可见制品发生字节变化，都不得回写 `logic-001`；该轮次保留并标记失效，修订材料进入 `logic-002`。正式派发前再把状态改为 `collecting`。

## 流程彩排

正式材料制作前先执行不含真实游戏、锚点或四项夹具的**流程彩排**。彩排制品保存在 `research/calibration-tests/logic-pilot/rehearsals/rehearsal-001/`，不放入正式 `runs/`，不使用四种理论结论，也不计入第二轮模型证据。

彩排的两个空白会话均使用 `gpt-5.6-sol`、`high`。只有以下九项全部满足，流程才通过：

1. 冻结输入与派发输入字节一致；
2. 两个空白会话收到同一输入；
3. 两份**首次提交**彼此不可见；
4. 提交原文与 SHA-256 原样保留；
5. 提交收齐前共享工作区没有明文真值或其他提交；
6. 揭示真值能够复算公开承诺；
7. 全部 JSON 通过对应 Schema；
8. Git 历史能够证明冻结、收集与揭示顺序；
9. 污染声明与异常全部保留。

任一项失败都保留 `rehearsal-001` 并明确报告流程失败；修正进入 `rehearsal-002`，不能覆盖或只展示成功版本。

## 人工放行门

流程彩排通过后，测试保管者可以继续制作、审核、校验并推送 `logic-001` 冻结包，但不得立即派发六项正式角色任务。正式派发前必须向作者展示冻结提交、制品清单、全部公开哈希与**答案承诺**，取得一次明确放行。
