# 0076 区分挑战结构与玩家相对难度

- 状态：已接受
- 日期：2026-07-21

## 决策

把**挑战结构（Challenge Structure）**作为规则侧可描述对象，记录目标或评价标准、可用动作与资源、信息条件、时间／空间／精度约束、随机或对抗因素，以及失败与重试后果。

把设计者或研究者关于所需认知、感知、操作、沟通和协调能力的判断写成**能力需求假设（Skill-Demand Hypothesis）**；把**玩家相对难度（Player-Relative Difficulty）**定义为特定玩家或群体在给定知识、能力、界面、设备、对手和环境下克服挑战的实际困难关系。二者都需要行为或体验证据，不能从规则结构直接断言。

## 影响

“简单／困难模式”作为规则集或配置标签不直接证明难度。案例应拆开记录参数、信息、窗口、失败成本等实际变化，并使用成功率、完成时间、尝试次数、错误、放弃、求助和主观报告等适当证据。不同指标可能冲突，沙盒中的玩家自选目标也可形成挑战结构。
