# 0054 以玩家活动连接机制与玩法模板

- 状态：已接受
- 日期：2026-07-20

## 决策

引入**玩家活动（Player Activity）**，描述玩家在与规则系统互动时反复进行的认知、感知、操作、沟通和协调活动。玩家活动与物理输入、规则动作和状态效果分开记录：规则可以接受一次输入并结算动作，但玩家为完成它而进行的听辨、预测、瞄准、规划、说服或手部控制不必各自成为正式规则。

玩家活动用于解释“机制——编排——>玩法模板”这条箭头，不加入“原语—机制—玩法模板—游戏”的公开主链。机制通过可能性、限制、信息、时机与反馈促成活动；编排把这些活动组织为持续或反复出现的玩法结构。

## 影响

案例分析应区分规则明确支持什么、设计者希望玩家做什么，以及玩家测试中实际发生什么。关于某机制会诱发某种推理、技巧、交流或协调的判断属于可检验假设，不能仅凭规则描述或作者意图当作已经成立的玩家事实。
