# 0053 区分玩家推理与规则机制

- 状态：已接受
- 日期：2026-07-20

## 决策

把**玩家推理（Player Inference）**首先视为机制促成的认知活动，而不默认视为正式规则机制。隐藏事实、约束、线索生成、信息访问、答案提交和验证属于规则侧结构；玩家依据观察进行比较、记忆、假设、演绎、归纳或概率估计，通常属于玩家侧活动。

当游戏要求玩家把判断外化为标记、声明、填写、指认或其他规则动作时，这些提交及其合法性检查、结算和反馈进入机制表达；产生判断的思维过程仍不因而自动成为规则。日常语言仍可使用“推理机制”，但正式分析必须展开其中的规则侧与玩家侧部分。

## 影响

逻辑解谜和社会推理案例应分别记录游戏提供、限制和验证了什么，以及玩家实际需要怎样处理信息。反复出现的“获取线索—更新判断—试探或提交—获得反馈”可以成为玩法模板中的活动循环，说明机制到玩法模板的编排需要考虑玩家认知，而不只是拼接规则结构。
