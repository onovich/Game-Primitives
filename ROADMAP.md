# 项目阶段约定

- 状态：已采纳
- 日期：2026-07-20
- 当前阶段：阶段一·开放研究

## 核心约定

《游戏原语》先作为研究项目生长，再成为一本书。

早期的主要任务是发现问题、积累证据、比较概念、拆解案例、提出和推翻假设。我们不把每次有价值的讨论立即改写成书稿，也不把现有目录当作必须依次完成的章节清单。研究材料保留探索过程；书稿则在研究逐渐成熟后，围绕读者的理解路径重新选择、组织和改写。

```text
讨论与观察 → 研究记录 → 模型与案例 → 综合论证 → 书稿
```

这不是只能前进的流水线。后续案例可以迫使我们返回前面的阶段，修改术语、语法或核心假设。

## 阶段一：开放研究

目标是扩大问题空间，而不是过早收敛。

主要活动：

- 进行主题讨论和思想实验。
- 阅读游戏研究、设计理论、形式语言及相邻领域资料。
- 拆解跨媒介、跨年代和跨类型的游戏案例。
- 收集反例、模糊案例和无法被当前模型解释的现象。
- 尝试不同的原语分类、规则句、机制表示与组合方式。
- 记录问题之间的关系，不急于给出统一答案。

这一阶段允许：

- 同一概念存在多个候选定义。
- 笔记重复、局部矛盾或暂时没有结论。
- 目录、术语和表示法频繁调整。
- 形成局部清晰的文章或页面，但不把它们自动视为书稿章节。

### 当前子周期：第一轮已关闭，第二轮待组样

项目仍处于阶段一。第一轮由 **4 个深度案例与 8 个标准案例**组成，已于 2026-07-22 在声明的结构范围内完成案例包、失败归并、跨案总汇报与 D-C1–D-C7 决策。具体范围与关闭边界见 [`research/calibration-cycle-1.md`](research/calibration-cycle-1.md)和[门 C 总汇报](research/calibration-gates/gate-c-first-cycle-synthesis.md)。

“第一轮完成”不表示模型完备，也不表示实际对局、复现、测量、玩家行为和体验研究已经完成；这些缺口继续保留在[校准证据积压](research/calibration-evidence-backlog.md)。

第二轮的方法已由[校准协议](research/calibration-cycle-2-protocol.md)确定：每个比较组必做**重构测试**，并从**设计变体测试／近邻辨别测试**中至少选择一项；涉及原语边界或行为主张时再触发相应条件检验；整轮按**反例优先采样**选择案例。样本组、研究问题和执行顺序仍待单独决定，协议获准不等于第二轮已经启动。

关闭一轮校准不等于进入阶段二；阶段转换仍按下文的证据与稳定性条件判断。

## 阶段二：模型整理

目标是把分散发现整理成可以比较和检验的模型。

主要活动：

- 明确术语层级、类型系统和分析尺度。
- 比较候选本体、语法和形式化记法。
- 用大量案例与反例检验机制分解。
- 建立可复用的案例模板、关系索引和证据链。
- 区分稳定结论、工作模型、争议与开放问题。

阶段转换不由日期或文件数量决定。当核心术语在多类案例中反复可用，主要分歧已经被明确表达，并且材料足以支撑连续论证时，才考虑进入下一阶段。

## 阶段三：书稿架构

目标是从“我们研究了什么”转向“读者应当怎样理解”。

主要活动：

- 确定核心读者、中心论点和全书承诺。
- 重新设计阅读顺序、章节职责和概念引入节奏。
- 从研究材料中选择证据，而不是把笔记顺次拼接。
- 编写样章，检验深入浅出、技术精度与叙事节奏。
- 确定图示、案例、练习、术语表和附录体系。

## 阶段四：完整成稿

目标是形成可以整体阅读、审阅和修订的书。

主要活动：

- 完成章节初稿与跨章节整合。
- 统一概念、符号、案例口径和中英术语。
- 进行事实核查、技术审阅、读者测试和多轮编辑。
- 处理引用、版权、插图、索引和出版形态。

## 阶段五：出版与正式网页版

目标是使书籍和网页成为相互补充的成熟作品。

正式网页版将根据稳定后的内容重新设计，而不是给早期研究网站换一层皮。它可以拥有不同于书籍的导航、交叉链接、交互图解、筛选器和案例数据库，但应共享经过核验的概念与内容来源。

正式内容优先按网页版编写、发布和阅读验证，充分利用共享术语、跨卷链接和案例关系；实体书与其他平台版本在同一内容源稳定后再作媒介适配。网页版优先不改变当前“先研究、后成稿”的阶段约定，早期 GitHub Pages 仍是研究阅读界面，而不是提前定型的在线书稿。

## 早期 GitHub Pages：研究版

早期网站的定位是**公开的研究阅读界面**，不是在线书稿。

它优先解决：

- 在浏览器中舒适地阅读现有材料。
- 按概念、机制、玩法模板和案例互相跳转。
- 搜索术语与开放问题。
- 看出内容是观察、来源、假设、项目定义还是暂定结论。
- 从页面回到对应的仓库源文件和版本。

实施时遵循这些约束：

1. **单一内容源**：尽量从仓库中的 Markdown 或结构化资料生成，不复制维护另一份正文。
2. **明确状态**：全站显著标为“研究版”，页面能够表示草稿、研究中、暂定、稳定或已废弃等状态。
3. **低迁移成本**：早期技术选型服务于阅读和维护，不追求最终视觉形态，不让框架锁定内容模型。
4. **研究型导航**：按问题、概念和关系组织材料，不必模仿书籍章节顺序。
5. **可追溯**：尽可能显示来源文件、更新时间或版本，使公开页面仍能回到研究上下文。
6. **允许重建**：进入成稿阶段后，可以更换技术、信息架构、文案和设计系统。
7. **善待旧链接**：最终重建时，为有外部引用价值的研究页面保留归档或重定向，而不是无提示消失。

网页的具体框架、目录、视觉设计和部署流程留到实际建设任务中决定。本约定只确定它的角色和生命周期。

## 当前判断

现在最有价值的成果不是“完成了多少章”，而是：提出了哪些好问题，找到了哪些有解释力的区分，积累了哪些可靠证据，哪些反例迫使模型变得更好。
