# 第一轮案例校准失败清单

- 状态：执行中
- 建立日期：2026-07-21
- 对应计划：[第一轮案例校准](calibration-cycle-1.md)
- 使用模板：[案例研究包模板 v0.2](../catalog/CASE-PACKET-TEMPLATE.md)

本清单汇总跨案例可比较的模型压力。案例研究包保留现场细节，这里只收录会影响共享术语、表达方法、取证方式或教学可读性的问题。记录失败不是给模型“扣分”，而是第一轮校准的主要产物。

## 记录原则

1. 先记症状和证据，后提修订；不能因为已有修订偏好而重写症状。
2. 一个案例中的不便不自动升级为全局问题；至少说明它为何可能跨案例复现。
3. 区分模型失败与资料不足、版本冲突、观察缺失、模板字段不合适。
4. 共享定义只在校准门集中修改，除非问题已经阻塞后续表达。
5. “没填出来”必须判定为未知、不适用、尚未取证或模型无法容纳之一。

## 失败类型

| 类型 | 判定问题 |
| --- | --- |
| 无法表达 | 当前词汇与关系是否根本无法表示一个有证据的结构？ |
| 表达冗长 | 虽能表示，是否复杂到失去比较或教学价值？ |
| 歧义 | 同一表达是否支持两种会影响结论的解释？ |
| 术语变义 | 同一术语是否在不同案例中偷偷改变操作性含义？ |
| 粒度漂移 | 比较对象是否在机制单元、复合机制、系统或模板之间滑动？ |
| 跨媒介失真 | 抽象是否忽略了软件、棋具、身体、界面或裁判的决定性作用？ |
| 因果越界 | 是否仅从规则推断了实际行为、策略、体验或价值判断？ |
| 证据不足 | 主张也许可表达，但当前来源不能支持？ |
| 版本冲突 | 不同规则、实现或配置是否被错误合并？ |
| 模板失配 | 字段是否持续重复、空白或迫使研究者虚构答案？ |

## 校准门 A：国际象棋 × Game Boy《俄罗斯方块》

### 共同能解释之处

- 两案都能沿“**原语**经有类型**语法**形成可运行**机制**，多个机制再由**编排**形成持续活动”的主线描述；无需增加公开层级。
- 输入、规则动作、结算、效果、反馈、世界／观察／信念，以及拥有／控制／许可等已有区分在跨媒介比较中保持有效。
- 机制尺度与编排图比“移动”“消除”等宽泛名称更能解释两案的相同与差异。

### 对照暴露的问题

- 动作不是单一瞬间；承诺、作出、完成、计时交接与系统确认可能分离。
- 时间结构需要表达运行机会和同一时点的调度，而不只是“回合制／实时制”。
- 效果可能改变实体身份与组成，不能把所有变化写成属性赋值。
- 规范、实装、执行与实际观察可能冲突，证据账本必须分层。
- 完整研究表达与读者可理解的最小表达需要同时保存并互相追溯。

### 拟议修改

| 修改对象 | 修改前 | 修改后 | 支持证据 | 代价与新风险 | 决定 |
| --- | --- | --- | --- | --- | --- |
| 动作表达 | 输入—动作—结果的层次，缺少中间承诺与完成边界 | 增加按需的**动作生命周期** | FIDE `made/completed`；Tetris 候选—回退—锁定 | 容易被误写成万能阶段表 | 采纳；ADR 0083 |
| 时间表达 | 时间作用域与触发策略 | 在时间结构内增加**调度语义** | Tetris 帧循环；棋钟与行棋许可双通道 | 研究充分视图字段增加 | 采纳；ADR 0084 |
| 效果表达 | 状态与关系更新 | 增加**实体身份效果**子类型 | 易位、升变、吃子与 Tetris 锁定 | 身份判断仍依赖声明尺度 | 采纳；ADR 0085 |
| 证据账本 | 规则／实现作为主张类型混合记录 | 增加规范、实装、执行、观察实现层 | FIDE 文本／裁判与 Tetris 手册／反汇编差异 | 不是所有非规则主张都适用 | 采纳；ADR 0086 |
