# 校准门 B：实际游玩观察资料审计

> 状态：观察资料包；不是规则资料包，也不是案例结论
> 核验日期：2026-07-21
> 对象：2026 WSOP Event #65 Freezeout 无限制注德州扑克；2016 英文版 *Agricola Revised Edition* 四人基础游戏
> 目的：区分“规则允许什么”“一局中实际发生了什么”“玩家自称在想什么”与“研究者据此作出的解释”

## 0. 精确匹配标准

本文件只把满足下列条件的材料当作本轮对象的直接观察资料。

### 0.1 WSOP Event #65

- 赛事必须是 2026 WSOP Event #65：`$1,500 Freezeout No-Limit Hold'em`；
- 观察必须能够回指该赛事，而不是同年其他无限注赛事或往届同编号赛事；
- 赛事结果、逐手报道、录像和参赛者证言分别登记，不能彼此冒充；
- 未公开暗牌、未记录的动作和玩家内心状态保持未知。

### 0.2 《农场主》

- 2016 英文版 *Agricola Revised Edition*，即本轮规则资料包冻结的基础盒；
- 四名玩家；
- 使用基础盒的职业牌与次要发展牌，每人直接获得 `7+7` 张；
- 不轮抽，不混入扩展牌、推广牌、周年盒新增内容、沼泽农夫或人数扩展；
- 若是数字实现，必须能够核验规则实现版本、卡池与全部设置，且不得默认等同于实体版。

“视频标题写着 Agricola”“四个人坐在桌边”或“规则大体相同”都不足以通过这些条件。

## 1. 观察证据不是一条单向等级

同一来源在不同问题上可能有不同地位。官方结果对赛事身份很强，却对逐手动作很弱；完整录像对动作顺序很强，却不自动证明玩家意图。为避免把“官方”误写成“什么都能证明”，本文件沿两条轴记录资料：

| 轴 | 要问的问题 |
|---|---|
| **身份权威** | 谁有权确认这是哪场赛事、哪个产品、哪些参赛者与什么最终结果？ |
| **观察完备度** | 资料记录了整局还是选段；是否保留动作、状态、时间、隐藏信息、裁定与最终结果？ |

并使用以下资料角色：

| 角色 | 含义 | 可用于什么 | 不可直接用于什么 |
|---|---|---|---|
| **E：事件记录** | 主办方或权利方的赛事、产品与结果记录 | 确认对象确实存在并发生 | 推断未记录的具体玩法 |
| **O：直接观察** | 现场记者、录像或可回放日志记录的一次实际发生 | 描述被记录的动作和可见状态 | 推断普遍频率、隐藏意图或设计因果 |
| **T：参与者证言** | 玩家或裁判对自己所见、所想的陈述 | 证明“此人如此陈述”，为意图假设提供线索 | 自动证明陈述准确、具有代表性或解释了结果 |
| **R：可复现轨迹** | 完整输入、动作序列、裁定与状态变化 | 重放并核查一局 | 代表所有玩家或常见玩法 |

当前检索获得了 Event #65 的 **E、O、T** 材料，但没有完整 **R**；没有获得严格匹配《农场主》四人局的 **O** 或 **R**。

## 2. 2026 WSOP Event #65

### 2.1 官方事件与结果记录

| 来源 | 身份与定位 | 可以支持 | 不能支持 |
|---|---|---|---|
| [WSOP 官方赛事结果页](https://www.wsop.com/tournaments/result/602/) | **E**；Event #65 最终结果 | 赛事于 2026 年 6 月举行；2,617 个参赛席位；Ciro Gonzalez 获胜；奖金与最终名次 | 任一具体牌局的完整动作、暗牌、筹码轨迹或策略 |
| [WSOP 官方赛事 Stories 页](https://www.wsop.com/tournaments/result/602/?tab=stories) | **E + 用户发布的 T**；内容寄存在官方赛事页，但帖文作者各自负责其陈述 | 该页确实把帖文绑定到 Event #65；Michael Gathy 在 6 月 24 日发帖声称赛事中存在拖延、叫钟与场务负荷问题 | 主办方认可帖文判断；拖延的总体频率；帖文所述手牌的完整轨迹 |

Stories 页不能因为寄存在 `wsop.com` 就整体升级成主办方裁定。Michael Gathy 的帖文是有身份线索的同时期参与者证言，适合成为后续查找录像、桌号或裁判记录的入口；在没有交叉验证前，不把它作为“赛事普遍存在拖延玩法”的单独证明。

### 2.2 PokerNews 现场逐手报道

[PokerNews 的 Event #65 现场报道入口](https://www.pokernews.com/tours/wsop/2026-wsop/event-65-freezeout/)包含三日的 Live Updates、筹码、奖金、图片与座次入口。其逐手条目由署名记者在赛事期间发布，属于外部媒体的**同时期直接观察**，不是 WSOP 规则正文，也不是机器生成的完整牌谱。它只覆盖记者选择记录的牌局。

#### 观察 O-H1：河牌加注后弃牌

[“Gonzalez Raises the River”](https://www.pokernews.com/tours/wsop/2026-wsop/event-65-freezeout/chips.836898.htm)记录了一手两人牌局的多街行动：

1. Gonzalez 在按钮位开到 750,000，Lin 在大盲跟入；
2. 翻牌后 Lin 过牌，Gonzalez 下注 475,000；
3. Lin 加注至 1,500,000，Gonzalez 跟注；
4. 转牌双方过牌；
5. 河牌 Lin 下注 2,500,000，Gonzalez 加注至 6,500,000，Lin 弃牌。

这足以证明该届赛事的一手被记录牌局中实际出现了**多街承诺升级、响应、延迟与未摊牌退出**。它不能证明 Gonzalez 在诈唬、Lin “读错了牌”、两人采用某个范围，或报道中的“施加压力”已经成为可观察的心理事实：双方暗牌没有公开。

#### 观察 O-H2：全下、跟注、摊牌与淘汰

[“Kyle Lin Eliminated in 2nd Place”](https://www.pokernews.com/tours/wsop/2026-wsop/event-65-freezeout/chips.836905.htm)记录 Gonzalez 在按钮位直接全下，Lin 跟注；双方公开 `A♣8♠` 与 `4♦4♥`，公共牌依次发出，Gonzalez 最终成三条并淘汰 Lin。

这足以观察到**全下承诺 → 对手接受 → 私有信息公开 → 随机牌序展开 → 手牌求值 → 赛事身份退出**的完整骨架。条目没有给出两人全下前的精确筹码、底池逐项构成、每个动作的秒级时间或实体牌序，因此仍不是可计算重放的完整牌谱。

#### 观察 O-H3：玩家自述的目标与风格

[决赛桌回顾及赛后采访](https://www.pokernews.com/tours/wsop/2026-wsop/event-65-freezeout/chips.836936.htm)记录 Gonzalez 自述：进入决赛日时先以进入决赛桌为目标，之后关注奖金级差；他也把自己描述为持续主动推进的激进玩家。

这属于 **T**：可写成“Gonzalez 在赛后如此描述自己的目标和风格”，并与逐手记录相互参照；不能改写成“所有玩家都以奖金级差为目标”“他每一手都很激进”或“激进风格导致胜利”。采访发生在胜利以后，还可能受到结果与叙事整理影响。

### 2.3 Event #65 目前可以与不可以声称什么

**有直接观察支持：**

- 至少有一手被记录牌局实际出现多街下注、加注、跟注、过牌和弃牌；
- 至少有一手被记录牌局实际出现全下、跟注、摊牌、公共牌展开、求值和淘汰；
- 一名获胜者事后明确陈述过阶段目标和自我认定的行动风格；
- 一名赛事参与者在同时期发帖声称观察到拖延与叫钟问题，但这一点仍需交叉验证。

**仍无足够支持：**

- 诈唬、读牌、范围构建、价值下注、底池赔率或 ICM 思考在某手中实际发生；
- 某类行动在整个赛事中的频率、成功率或代表性；
- 决赛桌全部牌局的完整行动轨迹；
- 玩家感受到的压力、乐趣、公平或紧张强度；
- 当前检索未找到该赛事决赛桌的官方完整转播、官方逐手牌谱或可下载重放日志。

因此，案例可以把“下注承诺会产生可观察的响应链”写成一次实际观察；把“诈唬”“读牌”“赔率计算”写成规则与结构所允许、文献常讨论但尚未在本案例中验证的**玩家活动候选**。

## 3. 2016 英文版《农场主》四人基础游戏

### 3.1 官方材料只能证明规则对象，不能证明一局发生

[Lookout 官方产品页](https://www.lookout-spiele.de/en/games/agricolare.html)、[2016 英文规则](https://www.lookout-spiele.de/upload/en_agricolare.html_Rules_Agricola-RE_EN.pdf)与[官方附录](https://www.lookout-spiele.de/upload/en_agricolare.html_Appendix_Agricola-RE_EN.pdf)能够冻结产品、人数、基础配置、阶段、卡牌与变体边界。它们是规范与产品资料，不是对局日志。

例如，规则说明四人基础游戏中哪些行动空间存在、每人获得七张职业与七张次要发展、工人如何排他占位以及何时收获。它只能支持“玩家**可以或必须**怎样行动”，不能单独支持“某位玩家实际抢了某格”“玩家感到喂养压力”或“某局形成了引擎”。

本轮查找了：

- 明确标为四人完整局、修订版完整局及高水平比赛的候选材料。

截至核验日，**未找到一份公开材料能够同时证明**：2016 英文修订版、四人、基础盒完整手牌、直接发牌而非轮抽、无扩展，并且保留可核查的完整对局行动。

阴性检索不能证明这种对局从未被录制，只能说明当前资料包没有找到可纳入证据账本的公开记录。搜索引擎未索引的私人录像、付费内容或需登录回放仍可能存在。

### 3.4 因此当前不应把哪些活动写成既成事实

在严格匹配观察出现前，本案例不能声称一局四人基础游戏实际发生过：

- 有意阻挡另一个玩家；
- 抢夺起始玩家、家庭增长或某个累积空间；
- 围绕喂养期限进行跨轮规划；
- 依据私人手牌建立引擎；
- 因他人占位而改变路线；
- 感到焦虑、紧张、满足或不公平。

规则材料可以证明这些行动中的一部分**可行、受激励或受约束**，但“规则结构足以激发某活动”仍应标作工作假设，而不是观察事实。

## 4. 补齐《农场主》观察缺口的最小方案

最可靠的近期方案不是降低版本标准，而是建立一份项目自己的受控四人对局轨迹。它将是一次 **O/R 级项目观察**，不是总体玩家行为研究。

### 4.1 对局前

1. 拍摄盒底、EAN `4-260402-315287`、英文规则版权页、全部 120 张基础牌与组件；
2. 清点 48 张职业、48 张次要发展、10 张主要发展和 14 张行动空间牌；
3. 明确拍摄没有扩展、推广牌、变体板块或其他语言/版本组件；
4. 记录四名玩家、座次、经验水平和起始玩家随机方法；
5. 记录 14 张行动空间牌在各阶段内的洗牌后顺序；
6. 直接向每人发七张有效职业和七张次要发展，保存发牌轨迹，并声明不轮抽。

### 4.2 对局中

每次放置至少记录：

```text
round / phase / actor / family member
chosen space / space state before placement
payment / immediate effect / optional branch
triggered card / resource state before and after
spaces now blocked / next actor
```

每轮另记累积空间的补入前值、补入量、领取者与领取后值；每次收获分开记录田地、喂养、乞讨、容纳与繁殖。保留一条连续广角录像，用局面照片和结构化日志互相校验。

### 4.3 玩家活动与意图

动作记录只能证明“选择了什么”。若要研究阻挡、抢位、喂养规划或引擎意图，在不打断正常行动的前提下记录简短即时口述，或在每轮末进行时间邻近的回溯提问：

- 你刚才优先考虑了哪些可选行动？
- 这个选择主要改变自己的路线，还是也考虑了其他人的需求？
- 哪条公开信息和哪张私有牌影响最大？
- 你预期哪个未来回合或收获期兑现这次投入？

把回答标作 **T**，不要让研究者用盘面替玩家补写动机。即使四人都说在“阻挡”，它也只证明这一局的证言，不证明四人《农场主》的普遍玩法。

## 5. 对校准门 B 的直接影响

两案当前的观察基础不对称：

| 问题 | Event #65 | 《农场主》精确四人基础局 |
|---|---|---|
| 对象与结果确实发生 | 有官方 **E** | 产品存在，但没有指定一局的 **E** |
| 具体行动实际发生 | 有选择性的 **O** | 未取得严格匹配 **O** |
| 玩家自述目标/意图 | 有少量 **T** | 未取得严格匹配 **T** |
| 完整可重放轨迹 | 无 **R** | 无 **R** |

因此，Gate B 可以用 Event #65 检验“规则可能”与“实际动作”必须分栏，也可以检验隐藏信息不能由记者叙述补全；但暂时不能用两个案例对称地比较实际玩家活动。

在《农场主》观察缺口补齐前，跨案例报告应采用以下措辞纪律：

```text
Event #65：被至少一个具体观察片段支持的活动
Agricola：由规则结构允许或激励的活动候选
两案共同玩法：尚待对称观察，不作事实性结论
```

这不是 Gate B 的失败，而是对“游戏制品”“规则实例”“实际游玩过程”和“玩家解释”四种对象进行分离后暴露出的真实资料缺口。
