- 建立日期：2026-07-21

- 使用模型：校准门 A 后共享词汇与[案例研究包模板 v0.2](../../catalog/CASE-PACKET-TEMPLATE.md)

本文在两份案例分析前登记问题、预期压力和判断标准。它不是结论，也不预先要求案例证明现有模型正确。

## 1. 为什么把这两个案例配成一组

两案都让玩家在多人顺序结构中提交稀缺机会，并根据他人的公开动作调整后续选择，但它们把信息、承诺和资源组织成明显不同的系统：

- 无限注德州扑克把私有手牌、公共牌、未知未来牌、下注历史、有效筹码和多人退出编排成多轮风险暴露；牌面评价只在部分路径的终局发生。
- 四人《农场主》把家庭成员、排他行动格、累积资源、私人卡牌、收获期限和终局计分编排成公开争位与跨回合发展；“取得资源”只是经济网络中的一个节点。

因此本组不用于证明“扑克和工人放置其实一样”，而用于检查当前词汇能否在保留差异的前提下表达二者。

## 2. 版本冻结前不得越过的边界

正式案例建档前必须分别锁定：

1. 德州扑克的规范来源、现金桌或锦标赛、桌上人数、盲注／前注、起始筹码、按钮与发牌方式、加注规则、all-in 与边池、摊牌和奇数筹码规则。
2. 《农场主》的出版方、语言、修订版印次或规则文件、四人基础配置、牌组范围、选牌方式、组件与勘误，以及是否排除扩展和单人／双人变体。

若权威来源之间存在冲突，先选择一个案例单元并把其他规范列为对照，不拼成一份“通用规则”。

## 3. 事前研究问题

### 3.1 信息

- **世界状态**、各玩家的**观察视图**与**信念状态**能否覆盖私有手牌、公共牌、未知牌序、弃牌、私人卡牌和他人意图？
- 查看、发放、揭示、亮牌、弃入 muck、公开行动与记忆历史，哪些是**信息访问操作**，哪些只是状态或呈现变化？
- “可以推断”是否需要区分规则可推导、概率可估计、对手模型与实际玩家已经推断？
- 信息在观察后是否持续可用、可记录、可验证或可遗忘，当前字段是否足够？

### 3.2 资源与经济

- 扑克筹码在玩家 stack、已下注区、主池和边池中分别具有何种所有权、控制权、保管和可分配性？
- 筹码同时承担行动上限、风险暴露、结算索取权与结果计量时，是否仍是一个资源，还是多个规则角色？
- 《农场主》的家庭成员、行动机会、房间、食物、谷物、动物、建筑材料和未领取的累积资源，哪些通过现有**资源准入**，哪些只是实体、容量、权限或计分事实？
- 两案的来源、存量、转移、转换、去向、刷新和反馈能否由当前**经济系统**定义表达？

### 3.3 机制与编排

- 德州扑克的发牌、下注、弃牌、all-in、边池、公共牌推进、摊牌与牌型评价，应在哪个尺度视为机制单元、复合机制或系统？
- 《农场主》的家庭成员放置、行动格占用、资源累积、建造、播种、繁殖、收获、喂养、家庭增长与计分如何划界？
- 现有编排边——顺序、并行、嵌套、门控、反馈、互斥、转换、共享资源／信息／时间窗、控制转移——是否足以表示：
  - 下注承诺如何改变后续合法动作与收益资格；
  - 某玩家退出后仍留在池中的资源；
  - 排他行动格如何阻断后来玩家；
  - 收获期限怎样让早期经济选择在之后结算。
- 若需要新边类型，它是否真有跨案例结构意义，还是现有边的参数或自然语言后果？

### 3.4 玩家活动与玩法模板

- “诈唬、读牌、价值下注、底池赔率”在何种证据下才可写为实际玩家活动、策略或涌现结构，而不是由隐藏信息直接推出？
- “阻挡、抢位、引擎构筑、喂养压力管理”在何种证据下才可写为实际活动，而不是规则提供的可能性？
- 两案各自的**玩法模板候选**能否用机制角色、持续活动、决策、反馈、目标连接和成立条件识别，而不退回“机制配方”或市场类型名？
- 单个机制是否足以形成局部玩法模板；若是，哪些编排关系其实发生在机制内部或机制与玩家活动之间？

### 3.5 校准门 A 修订的复验

- **动作生命周期**能否分别表达口头／动作下注的承诺边界、all-in、弃牌和工人放置的不可撤回性，而不强迫二者使用同一阶段名？
- **调度语义**是否对多轮下注、按钮位置、回合牌推进、阶段翻牌和收获周期提供净解释，还是与普通时间作用域重复？
- **规则实现层**是否能清楚区分规范规则、牌具实装、庄家／玩家执行与实际对局观察？
- **教学最小视图**是否仍能解释核心机制，同时把边池、奇数筹码、补充规则、卡牌例外和勘误留在研究充分视图？

## 4. 事前假设

以下均为可被案例否定的**工作假设**：

| ID | 假设 | 可能否证 |
| --- | --- | --- |
| B-H01 | 当前资源定义足以容纳筹码和农业资源，但必须明确位置、所有权、控制与用途，不能只标“数量” | 若同一筹码必须被拆成互不兼容的多个资源对象才能表达正常结算 |
| B-H02 | “承诺”首先是动作生命周期和权限／所有权变化，不需要立即新增顶层原语 | 若多个案例都需要无法由时间、关系和效果组合表达的稳定承诺操作 |
| B-H03 | 当前世界—观察—信念三分法足以容纳隐藏信息；主要缺口可能是观察的持续、验证与传播属性 | 若规则本身必须追踪一种既非世界事实、观察投影也非玩家信念的信息状态 |
| B-H04 | 玩法模板差异主要来自机制编排与持续玩家活动，而非各自拥有完全独有的原语 | 若两个案例只能靠题材名或不可复用的专有构件描述 |
| B-H05 | 诈唬、读牌、阻挡和引擎构筑必须保留证据状态，规则结构只能支持“可行／受激励”的有限主张 | 若规范文本直接规定并验证某项玩家认知活动，需重新划定规则与活动边界 |
| B-H06 | 模板 v0.2 能让核心循环保持可教，同时把边界规范和完整经济留在研究充分视图 | 若两个教学最小视图仍必须展示大部分完整规则才能不失真 |

## 5. 判定模型失败的条件

出现以下任一情况，应进入[失败清单](../calibration-failure-log.md)，而不是用更多散文掩盖：

1. 一个有充分来源支持的规则事实无法填入现有类型、槽位、时间、信息、资源或效果结构。
2. 同一共享术语必须在两案中改变操作性定义，且无法用角色、尺度、参数或关系解释。
3. 编排图只能列机制，无法表达决定玩法差异的跨机制传递和时序。
4. 为维持当前定义，必须把实际玩家行为伪装成规则事实，或把规则可能性伪装成已观察活动。
5. 教学最小视图删去某项后会改变核心机制含义，但保留它又使简明解释无法成立。
6. 规范、实装、执行和观察之间的冲突无法在证据账本中追踪。

表达较长、案例复杂或仍有未知本身不算模型失败；必须指出冗长来自模型结构，而非对象真实复杂度或资料缺口。
