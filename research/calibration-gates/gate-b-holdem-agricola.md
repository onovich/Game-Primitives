# 校准门 B：WSOP 无限注德州扑克 ×《农场主》修订版

- 状态：结构校准通过；D-B1–D-B6 已接受；行为证据待补
- 日期：2026-07-21
- 所属阶段：阶段一·开放研究，第一轮案例校准
- 重点：信息、资源、决策锁定、机制编排、玩家活动与来源术语

## 结论摘要

第二对案例继续支持保留公开主线：

```text
原语 --语法--> 机制 --编排--> 玩法模板 --实例化--> 游戏
```

它们没有要求增加公开层级，也没有提供足够理由把“下注”“工人”“承诺”“底池”或“阻挡”直接录取为新**原语**。真正需要澄清的是现有概念怎样跨角色工作：

1. **资源不是与实体、状态或关系互斥的名词类别，而是一种有准入条件的可叠加专业语义角色。** 筹码、家庭成员、动物、房间容量与行动机会都可能参与资源权衡，但必须说明载体、稀缺对象、竞争用途、资源操作、时间尺度，以及配置怎样改变未来可行行动。
2. **“承诺”不录取为新原语。** 下注与家庭成员放置都能用**动作生命周期**中的**锁定点**，加上状态、关系、资源、信息、许可与资格变化表达；项目把这种复合现象称为**决策锁定**，而不强行抽成一个动作名称。
3. **世界—观察—信念三分法仍然成立。** 当前模板的短板不在缺少第四种信息状态，而在没有稳定记录观察的保留、复查、验证、记录、传播、衰减，以及访问关系何时终止。
4. **来源自己的术语不能直接充当项目本体。** 《农场主》官方附录对 `resources`、`goods` 有内部分类；WSOP 对 `hand`、`action`、`pot` 也有赛事语境定义。来源术语只在**来源语域**、术语映射和元语言讨论中保留，项目分析改用具有独立身份的共享术语。
5. **“下注”和“工人放置”应作为术语族处理。** 表面名称可以分别指动作、机制、玩法模板或类型标签，但每个义项必须声明尺度与稳定身份；本案中的两个复合机制候选都不自动等于玩法模板。
6. **结构校准与行为证据必须分开判定。** 规则足以支持玩法模板的结构候选，却不足以证明诈唬、读牌、阻挡意图、最优路线或紧张体验已经发生。

本轮只调整共享定义的说明边界和案例模板，不改变四段主线。作者已逐项接受 D-B1–D-B6；模板升级为 v0.3，两份案例的结构校准通过，行为取证缺口继续保持开放。

## 2. 为什么这对案例有效

两案都让多人按顺序提交一种稀缺机会，而且已提交内容会限制后续选择；它们对机会、决策锁定和资源的组织却显著不同。

| 维度 | WSOP Event #65 NLHE | 四人《农场主》修订版 |
| --- | --- | --- |
| 案例尺度 | 一手牌嵌套于多日 Freezeout 锦标赛 | 一局 14 轮四人基础游戏 |
| 核心私有信息 | 两张暗牌；未知牌序；对手意图不属正式状态 | 每人 7+7 张起始手牌；阶段内行动牌顺序隐藏 |
| 公开状态 | 公共牌、下注、可见筹码、按钮与行动历史 | 行动格、累积物、已建农场、已打卡牌与轮次 |
| 稀缺提交 | 筹码投入、弃牌、全下与本手参与资格 | 每轮家庭成员放置、排他行动格与资源支付 |
| 提交后果 | 已投入筹码通常留在底池；弃牌退出索取资格；全下保留有限底池资格但失去后续下注能力 | 家庭成员占据格并立即结算，直到回家阶段前不能再用；格子本轮排他 |
| 资源网络 | 参赛者筹码在栈、下注区、主池／边池与获胜者栈之间重分配 | 多种物资生产、累积、储存、转换、建造、喂养、繁殖与计分 |
| 调度 | 按按钮和街道轮转；加注可重开行动；全下和弃牌改变跳过条件 | 准备—工作—回家—条件收获；顺时针多轮放置，人数不同会被跳过；卡牌可修改顺序 |
| 规则执行 | 荷官、Floor、WSOP 人员和参与者分布式执行／裁定 | 组件、规则书、卡牌文本与玩家共同执行，卡牌文字优先 |
| 持续反馈 | 筹码跨手保留，盲注按级别增长；单手信息与底池在手末重置 | 家庭增长增加未来行动，也增加喂养义务；农场和经济跨轮累积 |
| 终局评价 | 单手底池归属；赛事剩余最后一人 | 第 14 轮最终收获后按多个类别计分 |

若模型把两者都压成“花资源抢机会”，它将失去最重要的差异；若能用相同词汇表达承载形式、许可变化、时间边界和反馈，它才通过这轮资源校准。

## 4. 当前模型解释成功之处

### 4.1 **资源**定义可以跨载体工作

扑克筹码在玩家面前、下注区和底池中不是三种新物质。其规则意义由位置、控制、锁定状态、可退回性和底池索取资格共同决定：

```text
玩家筹码栈
→ 口头／筹码输入被认定为下注
→ 约束性输入与锁定投入
→ 主池／边池中的条件索取资格
→ 摊牌或仅余一人后的重新分配
```

《农场主》又表明，可计数并不自动等于同一种资源：

- 食物是储存、支付和喂养存量；
- 谷物可作种子、直接食物、烘焙投入和计分对象；
- 动物既是有容纳位置和繁殖条件的对象，也可在特定改良下承担转换资源角色；
- 家庭成员是持续实体，同时承载每轮可更新的行动容量；
- 房间是空间实体与容量提供者，不能只因限制家庭增长便整体改叫资源；
- 行动格是可寻址规则空间，其本轮使用许可可承担稀缺机会角色。

现有“**专业语义类型**位于核心语法角色之上”的结构允许这些角色叠加，不必建立“实体或资源”二选一分类。

### 4.2 **动作生命周期**容纳不同锁定点

扑克下注可能从口头声明或筹码动作开始，二者冲突时还有解释优先级；按顺序的声明可先于物理筹码移动变得有约束力，错误金额又可能在特定边界前被纠正。弃牌和全下分别改变本手资格与未来行动许可。

家庭成员放置则更短：选择未放置成员和未占行动格，提交位置，立即执行至少一个行动，成员与格子到回家阶段前保持占用。本轮放置许可在下一轮恢复，但行动产生的农场变化通常跨轮保留。

两者都存在**决策锁定**，却没有共享一条固定阶段名。这验证了校准门 A 的决定：**动作生命周期**应按案例命名里程碑，而不是推广一条万能流水线。

### 4.3 **调度语义**通过多人和规则修改器复验

德州扑克的一轮下注不是“每人行动一次”：加注可以使行动再次绕桌，弃牌和全下改变可行动集合，下注闭合后才推进公共牌或摊牌。锦标赛级别又只在下一手生效。

《农场主》的工作阶段同样不是固定四人轮询：每次只放一人，家庭成员较少者用完后被跳过；卡牌还可修改普通顺序。收获只在指定六轮开启，是条件阶段，不是十四轮都执行的固定尾声。

因此“回合制”依然只是粗标签；谁取得运行机会、何时跳过、何时重检和何时开启条件阶段，才是可比较结构。

### 4.4 世界、观察与信念没有偷换

两案都能分别记录：

```text
世界中实际是哪张牌／卡
≠ 某参与者能够看到什么
≠ 参与者记住或推断了什么
```

公开下注和占位动作可以改变他人的后续观察，却不直接把“对手很强”“某人准备抢位”等解释写入正式状态。现有**玩家推理**与**信念状态**边界通过本轮测试。

### 4.5 **实体身份效果**足以描述资格与增长

扑克通常不需要追踪每枚同面额筹码的长期个体身份；规则关注价值单位、容器和资格。暗牌实体在发出、亮牌、弃入 muck 和可能被追回之间有身份连续性边界。

《农场主》的家庭增长则把个人供应区中的成员转入在场家庭，并附着 newborn 条件；其本轮不行动、下轮开始增加行动容量。家庭成员、房间、田地和动物不能都退化成匿名数字。现有“保留、创建、移除、替换、重定型、拆分、合并”加状态／关系更新已能表达，不需新身份操作。

### 4.6 有类型编排图继续优于机制清单

德州扑克不是：

```text
抽牌 + 下注 + 比牌
```

而是私有发牌和公共揭示交替嵌套多轮下注；退出、全下与投入阈值持续改写行动者集合和底池资格。

《农场主》也不是：

```text
放工人 + 拿资源 + 建造 + 计分
```

而是每轮新增／刷新机会，玩家交替占用并立即结算，农场状态跨轮积累，收获期限阶段性结算生产、喂养与繁殖，家庭增长同时放大行动容量和义务。

这进一步支持用户最初的直觉：**原语到机制**可以高度语法化；**机制到玩法模板**更像有类型编排、反馈与成立条件，而不是另一套句法拼接。

## 5. 当前模型与模板的压力

### 5.1 来源术语与项目术语会发生词面碰撞

《农场主》官方附录把 crops 与 building resources 合称 `resources`，再把 resources、animals 与 food 合称 `goods`。项目的**资源**定义却按稀缺、替代用途和配置权衡准入，因而可能包含官方分类之外的食物、动物、容量或行动机会。

这不是翻译谁对谁错，而是两个系统在回答不同问题。若案例只写“资源”，读者无法判断一句话引用官方分类，还是使用项目分析术语。

WSOP 词汇也有相同风险：`hand` 可指一手流程、玩家所持牌、仍有效的牌或五张组合；`action` 既可指行动位置，也可指下注进展。来源词面不能自动决定项目的表达类型。

### 5.2 资源表缺少承载、容器和资格视图

模板 v0.2 已要求稀缺来源、竞争用途、操作和跨期影响，却可能诱导研究者把“筹码”写成一个平坦数值。边池必须同时表达：

- 数量来自谁的有效投入；
- 当前由何处保管；
- 哪些参与者仍有索取资格；
- 弃牌、全下、未跟注退回和摊牌怎样改变资格；
- 物理筹码身份与价值单位是否需要分别追踪。

《农场主》的累积格同样要求区分系统供应、行动格上的暂存量、玩家供应和田地／农场中的不可随意支付物。现有关系词汇能表达，但模板没有把这些关系聚到资源模块中。

### 5.3 信息表缺少观察后的命运

当前表格能写谁在何时看到暗牌或手牌，却不容易紧凑记录：

- 观察是否持续可访问，还是只在一个窗口出现；
- 是否允许记录、反复查看、请求核验或向他人传播；
- 弃入 muck、打出卡牌或公开牌面后，访问关系怎样改变；
- 历史观察仍存在于规则记录，还是只可能留在玩家记忆。

现有**信息访问关系**定义已经包含精度、延迟、持续性、可靠性和传播范围，因此这是模板显化不足，不是新本体层级。

### 5.4 “下注／工人放置”容易在机制与模板之间漂移

若把“放一个家庭成员”称为工人放置，它只是一项规则动作；若把“选择排他行动格并立即结算、每轮重置”称为工人放置，它是复合机制；若再把长期经济、期限和发展路线也纳入，它已经接近玩法模板或市场类型。

“下注”也可指一次 bet、完整 betting round、多街 wagering loop，甚至整个扑克玩法。案例必须声明尺度，不能让知名类型词替代运行边界。

### 5.5 玩家层仍缺对称且完整的行为资料

Event #65 已找到两个选择性现场观察：O-H1 记录多街下注、河牌加注后弃牌，O-H2 记录全下、跟注、公开牌面、求值与淘汰；另有冠军的少量赛后证言。它们能证明这些特定动作链发生过，却没有形成完整可重放牌谱。

严格匹配的《农场主》观察则仍缺失：公开候选分别混入旧版、周年版、数字卡池、单人配置或未说明 Draft/扩展，不能拼成一局不存在的“典型四人修订版”。规则书和赛事规则可以证明玩家被允许、要求或禁止什么，也可以记录出版方如何向玩家解释预期活动；它们不能证明：

- 玩家实际多久诈唬、怎样估计范围或采用何种下注尺度；
- 《农场主》玩家是否有意阻挡、怎样评价起始手牌或选择发展路线；
- 哪些结构对何种熟练度玩家造成紧张、满足或挫败。

保留这些空白是证据纪律通过，而不是案例失败。若深度案例必须同时完成行为研究，研究节奏又会被不必要地锁死。

### 5.6 官方规则对象本身也可能不完备

WSOP 赛事包没有在普通 NLHE 小节完整枚举标准 52 张牌和全部高牌型比较，也未明示 Event #65 的固定桌型；《农场主》官方下载没有基础盒 96 张手牌的完整卡面档案，而卡牌文字又优先于规则书。

这属于来源完整性与复现限制，不应被误判为理论无法表达。案例可以分析高置信核心结构，但不能宣称已经得到完整可执行规格。

## 6. 已采纳修订

### R1. 在案例包增加“来源术语映射”

作为按需研究表和写作约定，而不是新原语：

| 来源术语与来源身份 | 来源中的操作性含义 | 映射关系 | 项目共享术语或概念拆解 | 不能自动等同 | 定位 |
| --- | --- | --- | --- | --- | --- |
| `resource` | 某规则书内部分类 | 部分重叠 | 项目中的实体／内容类型，可能承担资源角色 | 项目的**资源**定义 | 页码／条款 |

来源术语只在带定位的原文引文、明确的规则转述、术语映射与元语言讨论中保留。映射允许等价、来源较窄、来源较宽、部分重叠、拆分和无对应项；完成映射后，项目分析行文使用共享术语。概念身份不依赖中英文、字体、颜色或粗斜体样式。

决定：采纳为案例模板可选模块和写作约定；不进入公开四段主线，也不进入原语词汇。见 [ADR 0088](../../docs/adr/0088-separate-source-and-shared-terminology-registers.md)。

### R2. 明确“资源是角色”，并补充资源关系视图

共享定义修订为：

> **资源**是可由实体、可互换数量、状态、容量、权限或机会承担的可叠加专业语义角色；候选必须说明稀缺、竞争用途或竞争者、可执行资源操作、时间尺度和对未来可行行动的影响。承担资源角色不会取消其原有核心语法类型，也不表示整个对象在所有机制中都是资源。

案例资源表增加最低准入字段：载体、稀缺对象、竞争用途、资源操作、时间尺度与未来行动影响；关系复杂时再展开承载形式／容器、所有／控制／保管、可用性与索取资格。它们复用已有**实体**、**状态**、**关系**和**规则权限**，不创造“资源账户”原语。

决定：采纳为定义澄清和模板字段；暂不拆出资源子本体。见 [ADR 0089](../../docs/adr/0089-treat-resources-as-qualified-stackable-semantic-roles.md)。

### R3. 显化信息访问的持续、验证与传播属性

模板的信息表增加一个组合字段：

```text
观察后的保留／复查／验证／记录／传播／衰减规则
```

它记录规则许可、实现或物质载体支持和游玩规范允许的能力，不声称玩家实际记住、相信、记录或传播了什么。

决定：采纳为**观察后效**字段；沿用带时间范围的**信息访问关系**，不新增“信息生命周期”层级。见 [ADR 0090](../../docs/adr/0090-record-time-scoped-information-access-and-observation-aftereffects.md)。

### R4. 不把“承诺”录取为新原语

机制卡继续使用：

```text
生命周期里程碑与锁定点
+ 状态与关系变化
+ 成本／资源操作
+ 撤回和纠正规则
+ 信息公开
+ 后续许可与资格变化
```

这些组合形成共享分析术语**决策锁定**。只有后续契约、竞价、秘密同时提交、外交或谈判等案例反复出现无法由这些结构表达的稳定操作，才重新提出原语候选。

决定：明确否决本轮录取；以**决策锁定**和**锁定点**描述并分解现象，同时保留“承诺”为来源术语、自然语言与跨案例检索词。见 [ADR 0091](../../docs/adr/0091-model-decision-locking-without-a-commitment-primitive.md)。

### R5. 用术语族管理“下注”和“工人放置”的定尺度义项

首次使用时必须声明至少一种尺度：

- 单个规则动作；
- 具有运行边界的机制单元；
- 多单元稳定耦合的复合机制；
- 包含持续活动和反馈的玩法模板候选；
- 市场／社群类型标签。

决定：不为两个表面名称规定唯一层级；以**术语族**管理具有明确尺度和稳定概念 ID 的义项，并在词汇和案例索引中保存组成、编排或实例化关系。跨层的是名称的行业用法，不是同一个设计对象。见 [ADR 0092](../../docs/adr/0092-use-term-families-with-scale-specific-senses.md)。

### R6. 允许“结构门审通过，行为证据待补”

两份案例可在玩家活动仍为“规则预期／研究假设”时完成本轮结构校准，只要没有把它们标作已观察事实。案例分别记录**结构校准状态**与**行为证据状态**；主张继续区分规则事实、结构推论、个案观察、跨案例支持和原型验证。

决定：本门记录为“结构校准通过；行为证据待补；理论结论仍可修订”。不让行为资料缺口阻塞进入标准案例，但不关闭相关行为失败条目；后续反例可以重开门审。见 [ADR 0093](../../docs/adr/0093-separate-structural-calibration-from-behavioral-evidence.md)。

## 7. 暂不决定

### 7.1 不建立资源实体树

可消耗、可再生、容量、机会、位置和信息都可成为设计上有用的资源描述，但本轮不足以证明它们应成为互斥子类型。优先记录可执行操作和关系。

### 7.2 不把诈唬、阻挡或引擎构筑写成正式机制

它们可以是玩家活动、策略或玩法模板候选；若规则没有要求和验证该认知活动，就不因经典案例广为人知而越权命名。

### 7.3 不把官方规则缺口自动补成项目规则

若为模拟器补齐牌组、牌型、洗牌、桌人数或完整卡面，应逐项标为项目夹具，并保留来源层级。缺口本身会继续指导取证。

### 7.4 不给两案宣称最终玩法模板名称

“多街隐藏信息下注”和“排他行动分配式长期发展”目前只是规则推导的结构候选。需与其他扑克、竞价、工人放置和德式桌游案例比较，才能确定模板边界与规范名称。
