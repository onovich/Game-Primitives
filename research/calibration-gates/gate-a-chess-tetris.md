# 校准门 A：国际象棋 × Game Boy《俄罗斯方块》

- 状态：待作者复核
- 日期：2026-07-21
- 所属阶段：阶段一·开放研究，第一轮案例校准
- 重点：规则语法、动作边界、时间模型、实体身份与案例研究包字段

## 结论摘要

第一对案例支持保留公开主线：

```text
原语 --语法--> 机制 --编排--> 玩法模板 --实例化--> 游戏
```

没有证据要求在公开主线中增加“规则”“动态”“体验”或“实现”等新层级。相反，案例说明主线要保持简洁，但箭头内部必须比当前模板更精确：

1. **动作不是一个无内部结构的瞬间。** 输入、候选动作、规则承诺、状态提交、完成、许可转移、计时交接和系统承认可能在不同节点发生。
2. **实时不是连续、回合不是离散。** 两案都由离散事件和阶段运行；真正的差别在谁推进时间、何时采样输入、怎样调度自动过程，以及行动许可是否等待玩家。
3. **实体效果不只改属性。** 一次机制可能同时更新多个实体，也可能创建、移除、替换、拆分、合并或重新定型实体。
4. **规则证据有不同实现层。** FIDE 的规范、实体执行和裁判裁量，与 Tetris 的说明书、程序实现和屏幕行为不能用一条“来源”字段压平。
5. **机制到玩法确实比原语到机制更不像句法组合。** Tetris 的玩法身份来自完整反馈闭环；国际象棋的玩法模板候选则必须等跨棋类对照，单凭机制清单不能稳定命名。

本轮建议只在共享模型内部增加四项表达能力，并改进案例模板；不改变四段主线，也不把任何新候选直接宣布为“游戏原语”。

## 2. 为什么这对案例有效

表面上，它们都很“简单”：一个棋盘、少量对象、清晰输入。结构上却几乎相反。

| 维度 | FIDE 国际象棋 | Game Boy Tetris A-Type |
| --- | --- | --- |
| 行动者 | 两名对手轮流选择；裁判有限介入 | 一名玩家局部操控；程序持续自动推进 |
| 规则来源 | 制定机构规范 + 物质执行 + 裁判裁量 | 玩家说明书 + 固定程序 + 硬件与呈现 |
| 时间 | 行棋许可交替，棋钟连续；`made` 与 `completed` 分离 | VBlank 同步主循环，输入采样、计时器、固定调用顺序 |
| 空间 | 8×8 地址空间，不同棋子读取不同可达关系 | 10×18 占据空间，复合形状姿态与碰撞 |
| 不确定性 | 无随机结算；对手选择与有限计算未知 | 系统生成未知块序；只预览一个未来块 |
| 状态历史 | 易位权、吃过路兵窗口、重复与 50/75 着 | 累积堆叠、行数、等级、随机器与多个短计时器 |
| 提交 | 玩家作出棋盘着法，通常再按钟完成 | 候选姿态逐帧接受/回退，触底后自动锁定 |
| 实体生命周期 | 棋子通常保持身份；易位同时更新两枚，升变替换类型 | 活动四格块锁定后解组为匿名固定占据 |
| 终局 | 将死、认输、和棋、超时或裁定 | top-out；A-Type 没有规则上的成功通关 |
| 玩法反馈 | 一手改变双方未来合法行动与局面评价 | 清行腾空间、加分并推动提速，直接反馈下一轮压力 |

若一个表达模型能在不改词义的前提下容纳这两者，它才有资格继续接受牌类、工人放置、实时战略和体育案例的压力。

## 3. 当前模型解释成功之处

### 3.1 有类型规则槽位有效

国际象棋普通棋步和 Tetris 平移都可以使用同一个检查顺序：

```text
输入或触发
→ 形成候选规则动作
→ 检查权限、前置与空间约束
→ 结算候选
→ 提交状态效果
→ 产生反馈
```

它们没有因此被宣布为同一机制。前者选择离散目的地并受王安全约束，后者只是对当前姿态作一格尝试且可能无效果回退。共同语法提供可比性，有类型内容保留差异。

### 3.2 状态、观察和信念必须分开

国际象棋的规则状态近乎公开，但玩家对局面的理解不相同；Tetris 又存在玩家可见 preview、不可见候选和可主动关闭的观察关系。现有“**世界状态—观察—信念**”模型不需改层级，已经避免两种常见错误：

- 完全信息不等于参与者知道最佳行动。
- 未来块未知不等于玩家的概率信念自动进入正式游戏状态。

### 3.3 历史摘要是必要状态

FIDE 的相同盘面可能因行棋方、易位权或吃过路兵机会不同而不是同一局面；Tetris 的相同可见堆叠也可能因当前块、preview、内部候选、等级与计时器不同而拥有不同未来。现有“**事件轨迹—历史摘要—游戏状态**”区分通过了这轮测试。

### 3.4 有类型编排图比机制清单有解释力

Tetris 不是：

```text
移动 + 旋转 + 随机 + 消行 + 计分
```

而是：

```text
发块
→ 在自动下降中操控
→ 碰撞锁定
→ 完整行检测
→ 清除并塌落
→ 计分、腾空间、提速
→ 再发块
```

顺序、门控、实体转换和反馈边共同决定其持续活动。这个案例直接支持“**玩法模板由编排特征识别，而不是由固定机制配方识别**”的工作定义。

### 3.5 四段主线仍有教学价值

两案都能沿主线讲清：先辨认规则表达的基本成分，再把它们组织成可运行机制；再看多个机制怎样形成持续活动；最后解释具体版本、媒介与配置如何使模板成为这款游戏。严谨性问题集中在箭头内部，不需要牺牲读者第一眼能懂的主线。

## 4. 当前模型失真之处

### 4.1 “一步”不是一个通用原子

FIDE 明确区分：

```text
触棋承诺
→ 棋盘着法 made
→ 对手取得行棋权
→ 通常按钟 completed
→ 棋钟运行方切换
```

这些节点通常紧邻，却并非同一事件。初稿曾错误地把许可转移和按钟完成合并；一手来源复核后已纠正。

Tetris 又是另一种边界：

```text
按键被采样
→ 候选姿态产生
→ 碰撞检测
→ 接受或回退
→ 多次成功变换累计成最终落点
→ 系统锁定
```

若把这些都写成 `Action → Effect`，表达虽然短，却无法说明承诺、失败无效果、响应窗口和计时责任。

### 4.2 当前时间模型缺“调度语义”

现有**时间结构**定义已经提到先后、并行、持续、延迟和响应窗口，但案例模板没有要求机制声明：

- 谁或什么时钟使它获得运行机会；
- 输入在何种频率或边界被采样；
- 与其他过程同一 tick 内的执行顺序；
- 是否允许暂停、积压、跳过或并发；
- 状态读取使用更新前还是更新后的快照。

这使 Tetris 的核心只能散落在“时间结构”“触发策略”和“编排图”三处。国际象棋也表明，行动许可通道与棋钟通道可以重叠而不同步。

### 4.3 效果需要表达身份变化

现有**效果**定义允许改变状态和关系，但模板容易诱导研究者只列属性赋值：

```text
Position(piece) := destination
```

王车易位需要一次提交多个实体的差分；升变涉及类型/实体替换；Tetris 锁定则终止活动复合实体并生成四个固定占据单元。缺少身份操作清单时，这些案例只能靠长篇自然语言补洞。

### 4.4 “规则是什么”取决于主张层

FIDE 有正式规范，但规范承认裁判需要处理未穷尽情形；Tetris 有面向玩家的说明书，却要靠固定二进制、反汇编和运行实验确定逐帧行为。两者说明至少要区分：

```text
规范或说明层
→ 实现/物质实现层
→ 执行与裁定层
→ 实际游玩轨迹
```

这些不是公开主线的新层级，而是规则主张的证据作用域。说明书“完整横行消失”与 Rev 1 程序仅扫描下方 16/18 行的差异，正是不能压平它们的具体理由。

### 4.5 深度案例的实现充分状态会淹没教学结构

国际象棋只为解释一次普通棋步，不必立刻展示非法着次数、申诉状态和记录表；Tetris 只为解释落块循环，也不必先列出所有内存计时器。反过来，边界分析又确实需要这些状态。

因此同一案例应提供两个可追溯视图：

- **教学最小视图**：刚好足以解释当前机制与设计问题。
- **研究充分视图**：足以复现合法性、边界和实现行为。

两者不是两个真相；前者必须链接自己省略的后者。

## 5. 建议修订

### R1. 引入“动作生命周期”，但不规定一条万能流水线

建议工作定义：

> **动作生命周期（Action Lifecycle）**：一次输入、动作或自动过程从被规则识别到退出运行边界之间，由规则承认的一组有序或分支里程碑；每个里程碑可以改变承诺、撤回权、成本、行动许可、计时责任、响应机会、状态可见性或完成判定。

案例必须按需命名里程碑，不把 FIDE 的 `made/completed` 强行推广为所有游戏术语，也不要求所有动作都有“声明—执行—完成”三段。它补充现有**动作结果**：生命周期回答“何时走到哪一步”，动作结果回答“各层面产生了什么”。

推荐：采纳为共享术语；先作为机制卡可选字段，不宣布为原语。

### R2. 在“时间结构”下明确“调度语义”

建议工作定义：

> **调度语义（Scheduling Semantics）**：规则决定动作、事件与过程何时获得运行机会的结构，包括驱动来源、更新频率或边界、阶段与优先级、同一时点的排序/并发、读取快照、暂停与积压，以及运行后何时重新检测。

它与**触发结算策略**的边界：调度语义安排所有动作/过程何时运行；触发结算策略处理多个已满足触发的规则实例怎样竞争与结算。简单回合制案例可以把大量调度信息交给回合作用域，实时/自动系统则必须展开。

推荐：采纳为**时间结构**的专业语义结构，不增加公开层级，也暂不直接准入原语表。

### R3. 给“效果”增加实体身份操作

正式效果按需声明：

```text
preserve / create / remove / replace / retype / split / merge
```

建议称为**实体身份效果（Entity-Identity Effect）**，定义为：效果对实体存在、类型、组成与连续追踪关系作出的规则变化。普通属性修改默认 `preserve`；王车易位是对两个保留身份实体的联合状态更新；升变需要说明逻辑身份是 retype 还是 replace；Tetris 锁定是活动复合实体 remove/split，并将四格物化为固定占据。

推荐：采纳为效果子类型；不把七个操作各自升格为顶层原语。

### R4. 给证据账本增加“规则实现层”

建议新增一个研究标记，而非本体层级：

| 实现层 | 回答的问题 |
| --- | --- |
| 规范/说明（specified） | 文档要求或向玩家承诺什么？ |
| 实装/物质化（implemented/materialized） | 程序、器材、场地和界面实际允许或阻止什么？ |
| 执行/裁定（enforced/adjudicated） | 谁发现、修正、解释和确认结果？ |
| 游玩发生（observed in play） | 某次或某组实际游玩发生了什么？ |

同一主张可以跨多层一致，也可以显式冲突。设计意图与玩家体验仍另按现有证据类型记录，不塞进这四层。

推荐：采纳为证据账本字段；不把 implementation 插进“原语 → 机制 → 玩法模板 → 游戏”。

### R5. 案例包改为“双视图 + 可选专业模块”

模板 v0.2 建议：

- 先给教学最小的规则世界和一张核心机制卡；
- 深度附录再给实现/裁定充分状态；
- 机制卡新增“动作生命周期、调度、执行来源、裁定权、实体身份效果”；
- 规则世界新增“规范—实装—执行—观察”的证据对应；
- 时间结构显式检查 tick/阶段/优先级/暂停；
- 资源、集合、规则修改器、状态附着等按需模块不强制展开；
- 版本卡增加来源文件 hash 与复现状态，但不在公开仓库保存受版权保护制品。

推荐：采纳。模板继续叫案例研究包，不因字段增长创造新的研究产物类型。

## 6. 暂不决定

### 6.1 不录取新的原语

`许可`、`历史窗口`、`可达关系`、`tick/计时器`、`候选—接受/回退`、`实体解组`和`观察开关`都出现了设计价值，但其中有些已经可由现有类型组合，有些只受一个案例支持。它们先留作候选，等后续扑克、农场主、Factorio、足球等案例检验跨案例复用和语义不可约性。

### 6.2 不在门 A 扩大“资源”

棋子、棋钟时间、空间容量和操作窗口都显示资源角色，却有截然不同的可储存、可转移与分配性质。德州扑克和《农场主》才是资源/经济的主压力案例，因此本轮只保留问题，不修改资源定义。

### 6.3 不把玩家活动写成已观察

规则足以建立“预期需要局面比较”“预期需要空间规划与时机操作”等能力需求假设，却不能证明策略采用、难度分布和体验。本轮没有为了让案例看起来完整而引用泛化二手攻略；真实行为材料进入独立观察批次。

### 6.4 不确定国际象棋模板的规范名称

Tetris 的编排闭环已经足以提出“实时落块堆叠—消行”结构候选；国际象棋的“交替式公开状态位置对抗”仍可能只是描述句。等围棋、将棋、战棋或抽象棋类对照后再决定其不变量与名称。

## 7. 案例研究包 v0.1 的表现

| 检查 | 结论 |
| --- | --- |
| 哪些字段持续有用 | 范围卡、通俗摘要、规则世界、机制尺度、编排、证据边界、失败、设计变体 |
| 哪些字段重复 | 来源说明、证据账本和范围卡有少量重叠；可通过共享来源矩阵链接而非删除追溯性 |
| 哪些字段太强制 | 资源、集合、规则修改器、状态附着在某些案例只能反复写“不适用”；应改成按需模块 |
| 哪些字段缺失 | 动作生命周期、调度语义、执行/裁定者、身份效果、复现状态、教学最小/研究充分双视图 |
| 是否过长 | 两个深度案例各约一篇长论文；作为研究包可接受，作为读者入口过重，必须有摘要视图 |
| 是否应拆成两个文件 | 暂不。一个研究包保留完整关系；网页可按视图折叠，书稿未来另写 |

## 8. 门 A 建议决定

| 编号 | 建议 | 理由 | 若不采纳的代价 |
| --- | --- | --- | --- |
| D-A1 | 采纳 R1–R4 为共享工作定义 | 两案都有直接结构压力；不改变公开主线 | 后续实时、卡牌优先权、体育裁判和实体转换会反复靠散文补洞 |
| D-A2 | 按 R5 把模板升级为 v0.2 | 属可逆研究工具调整 | 后续十案产生大量重复和字段漂移 |
| D-A3 | 保留国际象棋 `90+30` 仅作研究夹具，不称 FIDE 默认 | 可复现实钟，又不制造规范事实 | 不固定时限则无法完成计时实验；称默认则失实 |
| D-A4 | 保留 Tetris World Rev 1 作为候选目标，直到合法 dump 闭合证据链 | 当前来源足以高置信定位，但不够伪装为自有实测 | 改成模糊“1989 版”会混入 Rev 0 和后续包装 |
| D-A5 | 玩家行为观察与 ROM/实体复现不阻塞进入校准门 B | 当前门的目标是表达与证据边界，不是穷尽两款游戏 | 等全部实验完成会冻结研究节奏；忽略缺口则会过度声称 |

上述五项均有推荐默认，不需要改变项目方向。若作者无异议，执行顺序为：更新共享词汇与模板 v0.2 → 回填两案的新增字段 → 把未完成实验留在证据缺口 → 启动德州扑克与《农场主》的来源冻结和案例分析。
