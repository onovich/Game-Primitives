# 流程彩排任务

本任务只验证证据链，不测试游戏理论。

对输入中的每个项目：

1. 原样复制 `token`；
2. 计算 `left + right`，写入 `sum`；
3. 把 `confidence` 设为 `low`、`medium` 或 `high`；
4. 在 `notes` 中记录歧义；没有歧义时使用空字符串。

必须遵守：

- 不调用任何工具，不读取文件，不浏览网页，不联系其他执行者；
- 只使用派发消息中直接提供的说明、输入和提交模板；
- 按输入顺序返回两个 `responses`；
- 最终只输出一个 JSON 对象，不使用 Markdown 代码围栏或额外说明；
- 对象键按 Unicode 码点升序排列，使用两个空格缩进；保管者保存时只追加一个末尾 LF，不重排或改写内容；
- 如发生意外暴露或额外访问，必须如实写入 `pollution`，不能为了通过彩排而隐瞒。

每名执行者会在派发消息中收到各自的 `submission_id`、`submitted_at`、测试者标识、输入 SHA-256 和完整 JSON 模板。
