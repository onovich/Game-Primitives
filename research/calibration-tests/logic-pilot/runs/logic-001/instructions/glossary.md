# `logic-001` 测试本地术语表

本表只帮助读取结构化材料，不提供案例身份、案例角色、预期标签或项目结论。执行者可以使用不同措辞；判断以结构含义为准。

## 引用类型

- **实体**（`entity`）：规则可以区分和引用的对象或代理。
- **状态**（`state`）：对象、关系、规则集合或世界在某时刻可读取的值。
- **关系**（`relation`）：两个或多个引用之间有方向、角色或作用域的联系。
- **条件**（`condition`）：对事实、状态、关系或历史作出的布尔判断。
- **动作**（`action`）：代理可调用或系统可执行的操作。
- **过程**（`process`）：随时间或重复求值而展开的系统过程。
- **规则**（`rule`）：在声明作用域内把条件、行动与结果连接起来的规范结构。
- **目标评价**（`goal_evaluation`）：对完成、终止、得分或其他评价的判断。
- **反馈观察**（`feedback_observation`）：系统向某个观察者暴露的变化或信号。
- **上下文作用域**（`context_scope`）：限定规则适用的时间、空间、阶段、权限或信息范围。

## 规则槽位

`context`、`trigger`、`preconditions`、`actor`、`target`、`action`、`cost`、`resolution`、`effects`、`feedback` 与 `goals_evaluation` 分别记录规则作用域、进入求值的条件、执行前约束、负责者、直接对象、操作、承诺成本、结算方法、状态结果、可观察信号与评价更新。

## 组合关系

- `sequence`：前一结构的结果成为后一结构的输入或触发。
- `parallel`：多个结构在同一作用域内同时求值或争用。
- `nesting`：一个结构只在另一结构的阶段或上下文内有效。
- `gating`：一个结构决定另一结构是否可用或生效。
- `feedback`：结果改变后续同类结构的参数或输入。
- `mutual_exclusion`：一项成立或被选择时关闭另一项。
- `conversion`：同一承载物在两个结构中改变语义角色。
- `coupling`：多个结构因共享状态而产生非局部影响。

## 不确定与范围状态

- `encoded`：材料已明确表达。
- `not_applicable`：该槽位对当前规则没有语义内容。
- `source_underdetermined`：冻结材料不足以确定。
- `representation_gap`：材料明确，但当前表示无法承载。
- `scope_excluded`：事前排除在任务范围外。
- `indeterminate`：执行者依据当前输入无法作出唯一判断；必须说明缺少什么。

## 标注答案

- `yes`：题面规则和状态足以肯定问题。
- `no`：题面规则和状态足以否定问题。
- `indeterminate`：题面不足以肯定或否定；不得用作品常识补全。

`confidence` 只记录诊断信息，不进入通过判定。
