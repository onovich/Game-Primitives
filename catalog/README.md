# 案例目录

这里保存原语、机制、玩法模板和完整游戏的结构化分析。案例不是为了给作品贴唯一标签，而是为了检验理论能否解释差异、变体和反例。

## 建议记录格式

```markdown
# 名称

- 层级：原语 / 机制 / 玩法模板 / 游戏
- 别名与英文名：
- 分析尺度：
- 一句话定义：
- 玩家面对的核心问题：

## 结构

- 实体与角色：
- 相关状态：
- 前置条件：
- 动作或规则：
- 成本与结算：
- 效果与反馈：
- 时间结构：
- 空间结构：
- 信息结构：
- 随机性：
- 目标与终止：

## 组合

- 依赖的原语或机制：
- 组合关系：顺序 / 并行 / 嵌套 / 门控 / 反馈 / 互斥 / 转换 / 耦合
- 常见上层玩法模板：

## 证据

- 典型实例：
- 变体：
- 反例与边界：
- 来源：
- 开放问题：
```

同一个名称在不同分析尺度下可能指向不同对象。记录时必须写明尺度，不以目录位置代替论证。
